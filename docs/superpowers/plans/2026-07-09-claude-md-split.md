# CLAUDE.md Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Shrink `CLAUDE.md` from 1866 lines (~45k tokens loaded every session) to under 300 lines of rules plus a routing index, moving the extracted detail verbatim into four topic documents, and guard the result with a line budget enforced by `make check`.

**Architecture:** Pure text move. Four new documents (`docs/PACKAGES.md`, `docs/FRONTEND.md`, `docs/API.md`, `docs/OPERATIONS.md`) are generated by `sed`-extracting exact line ranges from the pristine `CLAUDE.md` at commit `deab0a5`, so fidelity is guaranteed by construction and verifiable by `diff`. `CLAUDE.md` is then rewritten to contain only rules, a `Kde co najdeš` routing table, and a one-line-per-package `Mapa balíčků`. A new `docs-budget` Make target fails the build if `CLAUDE.md` ever exceeds 300 lines.

**Tech Stack:** Markdown, GNU `sed`/`diff`, GNU Make, POSIX shell. No new dependencies.

**Spec:** `docs/superpowers/specs/2026-07-09-claude-md-split-design.md`

Establishes the guard before the fix, and proves it actually catches an oversized file. The target is **not** wired into `check` yet: it must fail today, and a red `make check` would block the intermediate commits.

- [ ] **Step 1: Write the guard target**

Append to `Makefile`, immediately after the `check:` target:

```make
## docs-budget: Fail if CLAUDE.md grew beyond its rules+index budget.
docs-budget:
	@lines=$$(wc -l < CLAUDE.md); \
	if [ "$$lines" -gt 300 ]; then \
	  echo "CLAUDE.md má $$lines řádků (limit 300). Detaily patří do docs/."; \
	  echo "Popisné detaily patří do docs/PACKAGES.md, docs/API.md, docs/FRONTEND.md nebo docs/OPERATIONS.md."; \
	  exit 1; \
	fi
```

Note the tab indentation — Make requires literal tabs, not spaces. The `$$` escapes `$` for Make so the shell sees `$lines`.

- [ ] **Step 3: Verify move fidelity — the bodies must be byte-identical**

`body()` prints everything after a marker, up to the next marker or EOF. Each `diff` must print nothing and exit 0. **These diffs are the authority on fidelity** — if a line count elsewhere disagrees, trust these.

```bash
SRC=/tmp/claude-orig.md
body() { awk -v m="$2" '
  $0 == m { f = 1; next }
  /^<!-- BODY / { f = 0 }
  f { print }
' "$1"; }

diff <(sed -n '24,991p'    "$SRC") <(body docs/PACKAGES.md '<!-- BODY BEGIN -->')  && echo "PACKAGES OK"
diff <(sed -n '992,1465p'  "$SRC") <(body docs/FRONTEND.md '<!-- BODY BEGIN -->')  && echo "FRONTEND OK"
diff <(sed -n '1504,1752p' "$SRC") <(body docs/API.md      '<!-- BODY BEGIN -->')  && echo "API OK"
diff <(sed -n '1466,1503p' "$SRC") <(body docs/OPERATIONS.md '<!-- BODY CLI -->')    && echo "OPS/cli OK"
diff <(sed -n '1791,1822p' "$SRC") <(body docs/OPERATIONS.md '<!-- BODY CONFIG -->') && echo "OPS/config OK"
diff <(sed -n '1753,1772p' "$SRC") <(body docs/OPERATIONS.md '<!-- BODY MAKE -->')   && echo "OPS/make OK"
```

Expected: six `... OK` lines and no diff output.

Note the trailing-blank subtlety: `sed -n '1466,1503p'` ends at the last CLI line, and the generator emits `\n## Konfigurační…` — that leading newline becomes the blank separator line, which `body()` excludes because it stops at the next marker. If `OPS/cli` shows one spurious trailing blank line, that is the bug; drop the leading `\n` from the following `printf`.
