# Google Takeout and Apple XMP sidecars

A Google Photos export (Takeout) or an Apple Photos export carries its metadata **beside** the image, not inside
it. The JPEG that comes out of Takeout has often been stripped of EXIF тАФ the real capture date, the caption you
wrote, and the GPS all live in a `.json` file next to it. Import the folder naively and you lose all of it: every
photo lands with no date and no description.

Teach the importer to read those sidecars.

**Prerequisite:** the folder import (`kukatko import dir`). Confirm it exists; if it does not, that task has not
run yet тАФ do not build a parallel import path, stop and say so.

## What to read

### Google Photos (Takeout)

For a media file `IMG_1234.jpg`, Google writes a JSON sidecar next to it. **The naming is a minefield and has
changed across Takeout versions** тАФ handle all of these:

- `IMG_1234.jpg.json`
- `IMG_1234.jpg.supplemental-metadata.json` (the current form)
- truncated variants (Google caps the sidecar filename length, so long names get cut тАФ `IMG_1234.jp.json`)
- `IMG_1234(1).jpg` pairing with `IMG_1234.jpg(1).json` (the index moves)

Match sidecars to media robustly and **report any media file whose sidecar could not be matched**, and any sidecar
that matched nothing. A silent mismatch is how people lose a decade of dates.

Fields to carry over:
- `photoTakenTime.timestamp` тЖТ `taken_at` (Unix seconds, UTC). This is the important one.
- `description` тЖТ the photo's description
- `geoData` / `geoDataExif` (`latitude`, `longitude`, `altitude`) тЖТ `lat` / `lng` / `altitude`. Google writes
  `0,0` for "unknown" тАФ treat exact zeros as absent, not as a point in the Gulf of Guinea.
- `favorited` тЖТ the importing user's favourite (favourites are per-user in Kuk├бtko)
- `people[].name` тЖТ **do not create subjects or markers from this.** Google's people data has no face boxes, so it
  cannot become a marker. If it is worth keeping at all, keep the names in a metadata field; do not fabricate face
  assignments.

**Do NOT create albums from the export.** Takeout's folder structure and `metadata.json` album files are full of
auto-generated junk from the phone. The user explicitly does not want them. Import the photos and their metadata;
leave album membership to `--album` on the import command.

### Apple Photos (XMP)

Standalone `.xmp` sidecars next to the media. Read via exiftool (already a dependency): capture date, GPS,
`dc:description` / `dc:title`, `dc:subject` (keywords), rating, `Artist`/`Creator`. Apple also writes `.AAE` files
for edits тАФ ignore them, they are not metadata.

## Precedence

The file's own embedded EXIF is the primary source; the sidecar fills what EXIF does not have тАФ **except** for the
Takeout case where EXIF has been stripped, which is the common one. Concretely:

- A field present in EXIF wins over the sidecar **only if it is plausible**. Google's re-encoding sometimes leaves
  a bogus `DateTimeOriginal` equal to the *export* date. If EXIF's date is after the sidecar's `photoTakenTime` by
  more than a small margin, the sidecar wins тАФ it is the true capture time.
- Never overwrite something the user has already edited in Kuk├бtko. Import fills gaps.
- Record where the date came from in the existing `taken_at_source` field (a new value like `sidecar` is fine).

## Verification

- Unit tests over real fixture sidecars (write a few by hand from the documented Takeout shape, covering: the
  three filename variants, a truncated name, `geoData` of exactly 0/0, a missing `description`, a `photoTakenTime`
  in the past with a bogus later EXIF date).
- Integration test: import a fixture folder that mimics a Takeout export тЖТ dates, descriptions and GPS land on the
  photos; **no albums are created**; an unmatched sidecar is reported; a second run changes nothing.
- `make check` must pass.
- Document: `docs/OPERATIONS.md` (the import command's sidecar behaviour and its reporting), `README.md`
  ("import a Google Photos export without losing your dates and captions" is a headline).

## Out of scope

- Creating albums, people or face markers from the export.
- Any other export format (Flickr, Nextcloud, тАж).
