# Ken Burns Effect

The slideshow offers three effects today — fade, slide, and none — defined in `web/src/lib/slideshowSettings.ts` and animated from `web/src/components/slideshow/slideshow.css`. Add a Ken Burns effect: a slow zoom and pan across each photo for the length of its slide.

## Requirements

- Add "kenburns" as a selectable effect alongside the existing ones, in the same settings card, persisted the same way.
- The motion runs for the full duration of the slide. When the interval setting changes, the animation duration follows it.
- Zoom and pan direction vary between slides so the show does not feel mechanical, but derive deterministically from the photo's uid — replaying the same album must look the same.
- The image must cover the stage for the whole animation. Never reveal empty space at an edge.
- Honour `prefers-reduced-motion`: fall back to a static slide with no motion.
- Applies to images only. Videos keep their current behaviour.

## Verification

`make check` must pass. Add unit tests asserting that the animation duration derives from the interval setting, that the same uid yields the same motion parameters, and that reduced motion disables the effect. Update `docs/FRONTEND.md`.
