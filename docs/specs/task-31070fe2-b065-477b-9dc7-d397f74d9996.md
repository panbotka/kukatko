# M7 — Mobile/tablet polish

A focused pass to make the whole app excellent on phones and tablets.

## Context
Read `docs/ARCHITECTURE.md` §1, §13 (full phone/tablet support). The app is react-bootstrap
(Superhero). This task refines responsiveness/usability across existing screens; it should not
change backend behavior.

## Requirements
- Audit and fix all primary screens (library grid, search, detail, albums/labels, people/cluster
  review, map, slideshow, upload, import admin, account) for small/touch viewports: tap target
  sizes, navigation (mobile navbar/menu), scroll/overscroll, image sizing, sticky toolbars,
  modals/sheets on mobile, and orientation changes.
- Ensure multiupload from the mobile gallery/camera is smooth; slideshow and detail work well in
  fullscreen on mobile; map gestures (pinch/zoom/pan) work.
- Verify no layout shift / horizontal scroll; safe-area insets respected.
- Keep i18n (cs/en) intact; no regressions to existing Vitest tests.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Add/adjust Vitest tests for responsive behaviors where testable (e.g. mobile menu toggles,
  conditional layouts via matchMedia mocks). Document any manual verification performed.
- Typed components; all text via i18n.
