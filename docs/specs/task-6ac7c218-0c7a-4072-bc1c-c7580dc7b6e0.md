# M1 — Multiupload UI (incl. mobile)

Build the upload experience: select or drag many files (including directly from a phone's
gallery/camera), upload with per-file progress and clear results.

## Context
Read `docs/ARCHITECTURE.md` §13 (multiupload incl. mobile). Backend: `POST /api/v1/upload`
returns a structured per-file result list (created/duplicate/error). react-bootstrap (Superhero),
i18n, fully responsive/touch-friendly.

## Requirements
- Upload page/modal: drag-and-drop zone **and** a file input that on mobile opens the gallery /
  camera (`accept="image/*"`, allow `capture` where appropriate, `multiple`).
- Queue many files; show per-file **progress** and final status (created / duplicate / error)
  with counts summary. Allow removing files before upload and retrying failed ones.
- Upload in batches/concurrently with a sensible cap; keep the UI responsive on large selections.
- Surface near-duplicate warnings returned by the API without blocking.
- After completion, offer a link to view newly uploaded photos in the library.
- All states/text i18n (cs/en); touch targets sized for mobile.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: queue add/remove, per-file progress + status rendering from mocked responses,
  duplicate/error handling, batch concurrency cap, retry. Mock fetch/XHR.
- Typed components; accessible labels.
