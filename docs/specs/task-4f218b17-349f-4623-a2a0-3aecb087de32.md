# Immersive photo viewer

Open a single photo into a full-screen, distraction-free viewer where the image owns the
screen and the chrome recedes — the way Apple/Google Photos and Immich present a photo.

## Requirements

- Opening a photo fills the viewport over a **near-black backdrop**; the image is centered and
  scaled to the largest fit without cropping.
- **Auto-hiding chrome:** the top action bar and any overlay controls fade out after a short
  idle and reappear on pointer move / tap. A persistent, unobtrusive close (Esc / ✕) always
  works and returns to the exact previous scroll position.
- **Keyboard navigation:** Left/Right arrows move to the previous/next photo in the current
  result set, Esc closes. Arrows also drive an optional on-screen prev/next control.
- **Smooth transitions:** fade/scale between photos and when opening/closing; respect
  `prefers-reduced-motion`.
- **Info panel:** a metadata panel (EXIF/date, location with a small map if present, people/
  faces, labels, album membership) slides in from the side on demand rather than always being
  visible — the default state is just the photo.
- Deep-linkable: the open photo and panel state live in URL params so Back and refresh behave.
- Use the design tokens for surfaces, the accent color and the focus ring; icons via the
  existing `Icon` component (bootstrap-icons only).

## Verification

- Drive the viewer in a browser against a throwaway instance: open, arrow through several
  photos, toggle the info panel, close, confirm scroll position is restored.
- `make check` must pass.
