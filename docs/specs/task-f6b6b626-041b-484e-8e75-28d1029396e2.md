# M5 — Import tracking schema

Add the table that records import/migration runs and high-watermarks for incremental imports.

## Context
Read `docs/ARCHITECTURE.md` §5.2 (`import_runs`), §9 (PhotoPrism import), §10 (photo-sorter
migration). External IDs (`photoprism_uid`, `photoprism_file_hash`, `photosorter_uid`) already
exist on `photos`.

## Requirements
- Migration creating `import_runs`: id BIGSERIAL PK, source TEXT CHECK(photoprism|photosorter),
  started_at, finished_at, status TEXT (running|done|failed), high_watermark TIMESTAMPTZ (e.g.
  max PhotoPrism UpdatedAt processed), counts JSONB (imported/updated/skipped/failed), last_error
  TEXT.
- Go `internal/importer` (or similar) repository: start a run, update counts, finish (done/failed)
  with watermark, and a query to fetch the latest successful watermark per source (for the next
  incremental run).

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): start/finish run lifecycle; latest-successful-watermark query
  returns the right value and ignores failed/running runs; counts update.
