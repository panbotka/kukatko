# Fix misaligned filter input boxes in the library

When the user opens the filters in the library, the input boxes are misaligned — inputs across adjacent columns start at different vertical offsets and don't line up in a clean grid. Fix the alignment so all filter controls line up.

## Where it is
- `web/src/components/library/FilterBar.tsx` (filter panel), with styles in `web/src/styles/app.css`.
- Two areas to check:
  1. **Advanced panel** (`AdvancedFilters`, the `Row.kukatko-filter-panel g-3`): the date-range column heads its inputs with a `<legend className="col-form-label pt-0 fw-semibold">`, while every sibling column (archived, has_gps, private, min_rating, flag, camera) heads its input with `<Form.Label className="small mb-1">`. The `col-form-label` legend has a different height than the `.small` labels, so inputs in adjacent columns begin at different Y positions. Mixed column spans in one wrapping row (a wide `lg={6}` date column with two stacked inputs next to several `lg={3}` single-input columns) with `g-3` gutters compound the misalignment on wrapped rows.
  2. **Facet row** (`FacetRow`): the Year cell is wrapped in a `Form.Group` with a `Form.Label`, while Album/Label use `SearchableSelect`, whose outer element is a bare `<div className="position-relative">` (no matching label wrapper) — a structural asymmetry. Heights are equalized via CSS min-height rules, but verify the labels/tops align.

## Requirements
- Make the filter controls align: labels of equal height across columns, and inputs sharing a common top baseline within each grid row.
- Preferred fix: normalize the headings so every column uses the same label style/height (e.g. replace the date-range `col-form-label` legend with a `.small mb-1`-style label consistent with the others, or give all columns equally-sized headings), and/or restructure the grid so columns in a wrapping row share a baseline. Keep accessibility (labels still associated with their controls; if you drop the `<legend>`, keep an accessible grouping/label for the date range).
- Keep it responsive across the existing breakpoints (`xs`/`md`/`lg`); don't break the mobile stacking.
- Do not change what the filters do — this is layout only.

## Verification
- This is a visual bug — verify visually. Run the dev server, open the library, open the filters (both the always-visible facet row and the advanced panel), and confirm inputs line up at the same breakpoints. Include the CSS min-height rules in `app.css` (around the `.kukatko-filter-facets` / `.kukatko-filter-panel` selectors) in your reasoning.
