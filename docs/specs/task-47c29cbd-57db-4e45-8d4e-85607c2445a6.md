# Job queue badges

An administrator should be able to see the state of the background job queue from any page, without navigating to a dedicated screen.

## Requirements

- In the right-hand side of the global footer, administrators see compact badges summarising the job queue: one badge per non-empty state (for example pending, running, failed) showing the state name and its count.
- Only administrators see the badges. For any other role nothing is rendered and no request is issued.
- Data comes from the existing admin-only job statistics endpoint, which returns counts grouped by state and by job type. Only the per-state counts are needed.
- The badges refresh on a modest interval of about 30 seconds, and polling pauses while the browser tab is hidden, resuming when it becomes visible again.
- A non-zero failed count is styled distinctly (danger styling) so it draws the eye.
- When every state is zero, a single quiet "idle" badge is rendered instead of a row of zeros.
- Badge labels are translated in `cs` and `en`.

## Edge cases

- A failing or slow request hides the badges silently rather than surfacing an error. The footer must never break a page.
- Polling stops when the component unmounts; no timer or in-flight request outlives it.

## Implementation notes

- If the global layout does not yet have a footer, add a minimal one to host the badges.

## Tests

- Component tests: nothing rendered and no fetch issued for a viewer, badges rendered from a stubbed response, the idle badge shown when all counts are zero, and no crash when the request rejects.
