# Migrate The Library To R2

Roughly 120 GB of originals sit on local disk and must move into the Cloudflare R2 bucket. This runs once, over hours, on a machine that may be interrupted. It must be safe to re-run.

## Requirements

- Add a CLI subcommand under the existing Cobra entrypoint, for example `kukatko storage migrate-to-r2`.
- For each photo in the catalogue: upload the original and its thumbnails to the bucket, verify the uploaded object (size, and the content hash where cheap), then update the row. Object keys are the `file_path` values already stored in Postgres, so no key assignment is needed. Only after the row is committed may the local file be removed.
- Removing local files is opt-in behind a flag, off by default. A dry-run flag reports what would move, how many bytes and how many objects, and nothing else.
- Idempotent and resumable. An interrupted run resumes rather than restarting, and a completed photo is skipped on a second run. Follow the high-watermark pattern the existing importers use (`internal/importer`).
- Never delete an original that failed verification. Never delete an original before its database row points at the object.
- Report progress: photos done, bytes transferred, failures, and an estimate of what remains. This job runs for hours; a silent process is a broken one.
- Bounded concurrency, configurable, defaulting low. Uploads must not starve a small VPS of memory or file descriptors. Stream each file; never read one wholly into RAM.
- Failures are collected and reported at the end rather than aborting the run, unless the failure is systemic (bad credentials, missing bucket), in which case stop immediately with a clear message.
- Note in the command's help text that R2 bills Class A operations for writes and includes one million per month free, so a full migration of roughly a hundred thousand objects is expected to be free — but a repeated full re-upload is not.

## Verification

`make check` must pass. Add an integration test against a real S3-compatible endpoint (MinIO is acceptable) that migrates a small fixture library, kills the run midway, resumes it, and asserts every object landed exactly once and no local file was removed for a failed upload. Document the command and its flags in `docs/OPERATIONS.md`.
