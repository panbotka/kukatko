# M6 — Optional Wake-on-LAN for the embeddings box

Optionally wake the GPU box via Wake-on-LAN when embedding/face jobs are waiting, so the queue
can drain without manual intervention.

## Context
Read `docs/ARCHITECTURE.md` §8 (auto-wake). The box is on the same physical LAN as the Pi; WoL
does **not** work directly over Tailscale (L3 overlay, no L2 broadcast) — the Pi sends the magic
packet on the local LAN. Use Go lib `mdlayher/wol`. **Default OFF** (manual power-on remains
fine).

## Requirements
- Config: `embedding.wake.{enabled(false),mac,broadcast_addr,interface,min_queue,cooldown}`.
- When enabled and there are pending `image_embed`/`face_detect` jobs above `min_queue` AND the
  sidecar health check reports offline, send a magic packet (with cooldown to avoid spamming).
  Re-check health after a grace period; if still offline, back off. Never block job processing on
  wake attempts.
- Log wake attempts and outcomes (sidecar became reachable or not). Do not attempt to power the
  box down (out of scope).
- Feature must be fully inert when disabled.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests: magic-packet construction for a MAC; trigger logic (only when enabled + offline +
  queue≥min + past cooldown); disabled → no packets ever. Use a fake packet sender (interface);
  do not send real network traffic in tests.
