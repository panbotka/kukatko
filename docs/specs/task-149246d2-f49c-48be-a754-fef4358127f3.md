# M5 — Import videos from PhotoPrism

Extend the PhotoPrism import to bring over videos (and live photos), not just still images.

## Context
Read `docs/ARCHITECTURE.md` §9 (PhotoPrism import). Depends on the PhotoPrism API client, the
PhotoPrism import job, and video ingest (media_type, poster, video metadata). PhotoPrism marks
media `Type` (image/video/live/animated/raw) and exposes files per photo.

## Requirements
- In the import flow, handle photos whose PhotoPrism `Type` is video/live/animated: download the
  correct **original video file** (and, for live photos, both the still and the motion file) via
  `/api/v1/dl/{hash}`, compute SHA256, store, and create a `photos` row with `media_type` set
  appropriately + probed video metadata (reuse the video ingest path).
- Generate poster + thumbnails for imported videos; enqueue embedding on the poster (consistent
  with uploaded videos). Map albums/labels/people as for images. Preserve `photoprism_uid` +
  `photoprism_file_hash` for dedup/incremental.
- Incremental + idempotent like the image import; per-item failures recorded, run continues.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB) with a **fake PhotoPrism** serving a video entry + tiny sample file:
  import creates a video photo with metadata + poster + external IDs and enqueues poster
  embedding; live photo links still + motion; re-run idempotent; incremental picks up new videos.
