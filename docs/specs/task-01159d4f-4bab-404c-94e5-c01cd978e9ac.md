# M3 — People UI

Build the people experience: browse subjects, review/assign face clusters, assign faces on a
photo, and clean up outliers — designed to be fast and low-friction.

## Context
Read `docs/ARCHITECTURE.md` §7, §13. Backend: subjects CRUD, `GET /photos/{uid}/faces`
(bbox+assignment+suggestions), assignment actions, `GET /faces/clusters` + cluster assign,
`GET /subjects/{uid}/outliers`. react-bootstrap (Superhero), i18n, responsive/touch.

## Requirements
- **People list page**: grid of subjects with cover, name, and photo counts; links to a subject
  page (their photos, edit name/type, set cover).
- **Cluster review page**: show unnamed face clusters; for each, a representative + sample faces,
  with a single action to **name the whole cluster** (assign to new/existing subject). Allow
  removing a stray face from a cluster before assigning. This is the primary, fast path.
- **Face overlay on photo detail** (component usable by the detail page): draw face boxes over the
  image using normalized bbox; click a box to assign/confirm a person, showing suggestions
  (one-click accept) and a free-text name field.
- **Outliers view** on a subject page: ranked suspicious faces with one-click unassign.
- All flows show progress and update optimistically; i18n (cs/en); usable on touch screens.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: cluster naming action calls the API and updates the list; face overlay positions
  boxes from normalized bbox and handles assign/suggestion-accept; outlier unassign flow. Mock API.
- Typed components; all text via i18n.
