# M6 — S3-compatible backup

Implement in-process, scheduled backups of the database dump and originals to an S3-compatible
bucket.

## Context
Read `docs/ARCHITECTURE.md` §14 (backup), §3.1. Use **`minio-go/v7`** with a configurable generic
S3 endpoint (works with AWS/MinIO/Backblaze/Wasabi), **path-style** addressing, and streamed
uploads (`objectSize=-1`, no full-file buffering). Config: `backup.s3.{endpoint,region,bucket,
access_key,secret_key,path_style}`, `backup.schedule`.

## Requirements
- `internal/backup` package:
  - **DB dump**: shell out to `pg_dump` (custom/compressed format) on the configured database,
    streaming the output to S3 (object key like `db/kukatko-<timestamp>.dump`). (Timestamp passed
    in / from the scheduler — do not rely on forbidden time calls in unusual contexts; standard
    `time.Now()` in app code is fine.)
  - **Originals sync**: upload originals to S3 incrementally (skip objects already present by
    key+size/etag), streamed.
  - Configurable **retention** (keep last N dumps / prune old) — documented and safe.
- **Scheduler**: run backups on `backup.schedule` (cron-like) from within `kukatko serve`; also a
  `kukatko backup` command + admin endpoint `POST /api/v1/backup` to trigger on demand, plus a
  status/last-run readout.
- Robust error handling and logging; never log secret keys; clear failure surfacing.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Tests against a **fake S3** (httptest or a minio-go-compatible stub): streamed upload with
  `objectSize=-1` + path-style; originals incremental skip logic; retention pruning; pg_dump
  command construction (skip actual dump if `pg_dump` absent in the unit suite); schedule parsing.
