# Show the full Library bulk-action bar on all photo-list pages

When photos are selected in the Library, a rich floating bulk-action bar appears
with the full set of batch actions. On the Album detail, Label detail, Favorites,
Search results, and person/subject gallery pages, selecting photos surfaces only
a stripped-down toolbar (often just a single "další úpravy" button), so the
common actions the user expects — add to another album, add/remove labels — are
missing. Bring the full bulk-action bar to every photo-list page, while keeping
each page's own context-specific actions.

## Current behavior (root cause)

- The Library renders `web/src/components/organize/BatchActionBar.tsx` — a
  `position: fixed` frosted bottom bar that slides up when ≥1 photo is selected.
  It is rendered ONLY by `web/src/pages/LibraryPage.tsx` (grep confirms a single
  usage). Its actions: clear selection, "Vybrat vše" (select all), "Přidat do
  alba" (add to album), "Štítky" (add + remove labels), "Oblíbené" (favorite),
  "Archivovat" (archive), `DownloadZipButton`, `StackSelectedControl` ("Seskupit"
  / group into a stack), and "Další úpravy" (opens the full `BulkEditModal`).
- Album detail, Label detail, Favorites, Search, and the person/subject gallery
  instead render `web/src/components/organize/SelectionBar.tsx` (a sticky-top
  toolbar) with far fewer actions. Album adds "Nastavit obálku" (set cover) +
  `BulkEditControl` + "Odebrat z alba" (remove from album); Label shows just
  `BulkEditControl`; the others are similarly minimal.

## Requirements

- On Album detail, Label detail, Favorites, Search results, and the
  person/subject gallery, when at least one photo is selected, show the same rich
  bulk-action bar the Library uses (`BatchActionBar` behavior): clear, select all,
  add to album, add/remove labels, favorite, archive, download zip, group/stack,
  and "Další úpravy".
- PRESERVE each page's context-specific actions by keeping them available on the
  same bar (merge them in — do not lose them):
  - Album detail: "Nastavit obálku" (set the selected photo as the album cover)
    and "Odebrat z alba" (remove the selected photos from this album).
  - Label detail: any label-specific action currently on its `SelectionBar`.
  - Person/subject gallery: any subject-specific action currently present.
- Only ONE bulk toolbar should be visible per page — do not stack the new bar on
  top of the old `SelectionBar`. Consolidate so the page shows a single coherent
  action bar. Retire the redundant `SelectionBar`/`BulkEditControl` path on these
  pages where the full bar now covers it, but keep the page-specific actions.
- The bar must only appear for writers and only while photos are selected, exactly
  as in the Library.
- Every action must operate on the correct selection and refresh the page's data
  on success (the `onEdited`/reload callback the pages already pass to
  `useBulkEdit`).

## Implementation notes

The cleanest approach is likely to make `BatchActionBar` accept optional
page-specific extra actions (e.g. a slot/children or a small typed list of extra
buttons) so Album can inject "Nastavit obálku" / "Odebrat z alba" and Label/
Subject their own, then render `BatchActionBar` on each of these pages the same
way `LibraryPage` does. You decide the exact API, but keep it typed (no `any`) and
consistent with the existing component conventions in `web/src/components/organize`.
