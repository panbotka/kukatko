### API â€” `internal/photoapi`

## Edge cases

- Existing rows: all new columns take their defaults (empty string / false). Nothing backfills them in
  this task; that is fine and must not be worked around.
- The `fts` rebuild must not lose existing search behaviour â€” add a test that searching by an existing
  title still works, plus one that searching by `subject` and by a keyword now matches.
