# Global footer

Every page gets a footer identifying who operates the system and linking to its source code. There is no footer today.

## Requirements

- The footer sits below the main content in the global layout component and therefore appears on every page that uses that layout.
- It states that the system is operated by SDH Veselice, and links to the project's GitHub repository at `https://github.com/panbotka/kukatko`.
- The link opens in a new tab with the safe `rel` attributes, and carries an accessible name. Any icon used is decorative and hidden from assistive technology, per the project's icon convention.
- Footer text comes from i18n in both `cs` and `en`.
- On short pages the footer neither floats nor overlaps content; it simply follows the content.
- The footer does not appear on the fullscreen slideshow, which renders outside the layout shell.
- The layout is structured so that a small status area can later occupy the footer's right-hand side without restructuring it.

## Tests

- Component test asserting the footer renders the operator text and a GitHub link with the expected `href`, `target` and `rel`, and that its icon is hidden from assistive technology.
