# Fullscreen photo viewer (lightbox) with left/right navigation

On the photo detail page (`web/src/pages/PhotoDetailPage.tsx`) the main preview is a `fit_1920` image. Clicking it should open a fullscreen lightbox with large previous/next controls so the user can browse photos immersively.

## Requirements

- Clicking the main photo image opens a **fullscreen overlay viewer** on a dark backdrop, showing the image as large as possible (contain), honoring the saved non-destructive edit (same CSS transform/filter the detail page already applies via `lib/photoEdit`).
- **Previous / Next** navigation with large, obvious arrow controls (left and right). Navigation follows the SAME order and scope the detail page already uses for neighbor navigation (`usePhotoNeighbors`, which pages the source `GET /photos` with the scope + filters from the URL). Behavior at the ends should match the detail page's prev/next (stop at ends).
- **Keyboard:** Left arrow = previous, Right arrow = next, Esc = close.
- **Touch:** swipe left/right to navigate on mobile.
- **Close:** a large close button (top-right, meets ~44px tap target) and clicking the backdrop both close the viewer, returning to the same photo's detail page.
- **Preload** the adjacent (prev/next) images so navigation feels instant.
- Keep the detail page URL correct so browser Back still works (either the viewer navigates the same route as the detail prev/next, or restores the current photo on close — "Back always works").
- **Videos / live photos:** clicking should open the appropriate fullscreen for that medium (native HTML5 video fullscreen via the existing `VideoPlayer`), NOT the still-image lightbox.
- Reuse existing fullscreen / slideshow patterns where sensible (`components/slideshow/Slideshow`, the Fullscreen API usage there) instead of duplicating logic. This viewer is a quick single-photo lightbox, distinct from the `/slideshow` auto-advance experience.
- Controls must be large and have visible focus states.
