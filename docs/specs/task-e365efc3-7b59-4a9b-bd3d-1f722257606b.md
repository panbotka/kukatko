# Dockerize Kukátko and publish images to GHCR

Add a container image build and a GitHub Actions workflow that publishes it to the GitHub Container Registry (ghcr.io). The app will run on an amd64 VPS. Model the approach on the sibling project photo-sorter, adapted to Kukátko (no LaTeX/font stack — Kukátko's runtime dependencies are the media shell-outs only).

## Requirements

### Dockerfile (new, at repo root)
- Multi-stage, producing a small static image:
  1. **Frontend stage** (`node:22-alpine`): `npm ci` and `npm run build` inside `web/`, producing `internal/web/static/dist`.
  2. **Backend stage** (`golang:1.26-alpine`, `CGO_ENABLED=0`): download modules, COPY the built `dist/` from the frontend stage into `internal/web/static/dist/` **before** `go build` (the frontend is compiled into the binary via `//go:embed`, so it must exist at build time), then build a single static binary.
     - Inject version info via `-ldflags`. The module is `github.com/panbotka/kukatko`; find the ldflags-injectable vars in `internal/version` (`Version`, `Commit`) and set them from build args `VERSION` and `COMMIT_SHA`. Use `-s -w` too.
  3. **Runtime stage** (`alpine:3`): install only the external tools the upload/media pipeline shells out to. Determine the exact set from the code and docs, but expect: `ffmpeg` (provides ffprobe too — video metadata/poster/transcode), `exiftool` (EXIF + XMP sidecar), `libheif-tools` (heif-convert for HEIC), and a RAW decoder (`libraw-tools`/dcraw — check `internal/imgconvert`). Also `ca-certificates` and `tzdata`. Do NOT install libvips — the thumbnailer defaults to the pure-Go engine.
     - Run as a non-root user, `EXPOSE` the app's default serve port (read the default from `internal/config`), `STOPSIGNAL SIGTERM` for graceful drain, `ENTRYPOINT` the binary with `CMD ["serve"]`.
- Add a `.dockerignore` so the build context excludes `.git`, `node_modules`, `web/node_modules`, `.secrets`, local config, and build artifacts.

### Publish workflow (`.github/workflows/docker-publish.yml`)
- Triggers: push to `main`, push of tags matching `v*.*.*`, and `pull_request` to `main` (PRs build only, never push).
- Registry: **ghcr.io**, image `ghcr.io/<owner>/kukatko` (use `${{ github.repository }}`). Authenticate with the built-in `GITHUB_TOKEN` (`packages: write` permission). No extra secrets.
- **The image must be built and pushed ONLY after the test suite passes for that ref.** Add a `test` job that runs the project's full tests (`make test`), and make the `build` job depend on it via `needs:`. The `test` job must mirror what the tests need — reuse the setup from `.github/workflows/ci.yml` (Go 1.26, Node 22, and for integration tests a `pgvector/pgvector:pg17` service + ffmpeg/exiftool + the `vector`/`unaccent` extensions and `KUKATKO_TEST_DATABASE_URL`). If tests fail, no image is pushed.
- Tagging scheme (use `docker/metadata-action@v5`):
  - Pushes to `main` → the **`latest`** tag (only on the default branch; do NOT tag `latest` on version-tag pushes).
  - Tag pushes `vMAJOR.MINOR.PATCH` → semver tags `{{version}}` and `{{major}}.{{minor}}`.
  - Also add an immutable `sha` tag.
- Build with `docker/build-push-action@v6`, passing `VERSION` and `COMMIT_SHA` build args (version = the git tag without leading `v` on tag builds, else `dev`; commit = short SHA). `push` is true only when `github.event_name != 'pull_request'`.

### `.env.example` (new, at repo root)
- A documented example of the environment variables needed to run the container on the VPS. Derive the variable names from `config.example.yaml` and `docs/OPERATIONS.md` (the `KUKATKO_` prefix convention, plus the `MAPY_API_KEY` exception). Cover at least: `KUKATKO_DATABASE_URL` (required), the embedding sidecar URL, storage/R2 keys, backup S3 keys, and `MAPY_API_KEY`. Use placeholder values, never real secrets. This file is an example and safe to commit (do NOT commit a real `.env`; ensure `.env` is gitignored).

## Edge cases / notes
- The frontend `dist` must be present before `go build` or the `//go:embed` will fail — verify the copy ordering.
- Keep the runtime image lean; only add packages the code actually shells out to. Confirm the package names exist in Alpine.
- Do not weaken or bypass `make check`.

## Done when

- `make check` and `make dev` pass; commit and push.
