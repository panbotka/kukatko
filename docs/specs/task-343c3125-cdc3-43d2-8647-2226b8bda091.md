# Global Search — Backend

Extend search to cover albums, labels, and people (subjects) by name/description, in addition to the existing photo fulltext, and expose a single grouped global-search endpoint for the navbar quick-results and the search page. The existing photo fulltext search (`GET /search`, `photos.Store.Search` over the `fts` tsvector) stays unchanged; this adds non-photo entity search and a grouped aggregator.

## Requirements
- Add case-insensitive, accent-insensitive name/description search to the existing stores:
  - `internal/organize`: `SearchAlbums(ctx, q, limit)` matching album `title`/`description`, and `SearchLabels(ctx, q, limit)` matching label `name`.
  - `internal/people`: `SearchSubjects(ctx, q, limit)` matching subject `name`.
  - Reuse the project's `immutable_unaccent` wrapper for diacritics-insensitive matching where practical (ILIKE over unaccented text is acceptable). Each method caps results at `limit`.
- New package `internal/globalsearchapi`: `GET /api/v1/search/global?q=` (RequireAuth) → grouped top-N results `{query, albums:[…], labels:[…], people:[…], photos:[…]}`. Photos reuse the existing fulltext search with a small limit. Cap each group (e.g. 5–8). Empty/whitespace `q` → 400. Each result carries enough to link and render: albums (uid, title, cover, photo_count), labels (uid, name, photo_count), people/subjects (uid, name, cover), photos (the usual photo shape).
- Mount via a new `server.WithAPI` builder in `cmd/kukatko/`, sharing the existing `organize` / `people` / `photos` stores. Use small interfaces so the handler is unit-testable.
- Integration tests: matching albums/labels/people/photos by name (include a diacritics-insensitive case), per-group limits respected, and empty query → 400.

## Done criteria
- Update `CLAUDE.md`/`README.md` (new `/search/global` endpoint and the new store search methods).
- `make check` must pass. Follow the project's strict lint and testing rules.
