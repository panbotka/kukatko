# Selective PhotoPrism import: --label, --person, --year

`kukatko import photoprism --album <uid>` already exists (commit 14ee9c7): it imports one album whole, maps just that album, and completes the run with a **nil watermark** so a later full import still sees every photo. Extend the same scoped mode to labels, people and years, so the library can be migrated in slices.

## Requirements

- New flags on `kukatko import photoprism`, each scoping the run the same way `--album` does:
  - `--label <slug>` — the photos carrying that label
  - `--person <name>` — the photos a named subject appears on
  - `--year <YYYY>` — the photos taken that year
- Flags **may be combined** (including with `--album`); they narrow the run together (PhotoPrism `q` terms are space-separated AND). No flag = the current full incremental run.
- **Every scoped run leaves the watermark untouched** — same rule and the same reason as `--album`. A test must assert it for each new filter, not just for albums.
- A scoped run maps the structure of the photos it imported (see the companion task on complete context transfer); it must not walk the whole source catalogue.
- `kukatko import photoprism --help` explains that scoped runs are partial and do not advance the resume cursor.

## Verified against the production instance

These search terms all work today on `https://fotky.kotrzina.cz` (PhotoPrism Plus 251130), each returning photos through `GET /api/v1/photos?count=…&merged=true&q=…`:

```
q=year:1985            q=label:"sdh"
q=person:"Aleš Kozák"  q=subject:"Aleš Kozák"     (both work)
```

`photoprism.PhotoListParams.Query` already passes `q` through verbatim, and `AlbumUID` maps to `s=`. Build the `q` expression from the flags; quote values that contain spaces (a person's name does).

## Implementation Notes

- `internal/ppimport`: `ImportAlbum` is the template — a `runState.albumUID` scopes the listing and `Complete(ctx, id, nil, counts)` withholds the watermark. Generalise it to a scope value (album uid + `q` terms) rather than adding three more near-copies.
- `cmd/kukatko/importcmd.go` holds the flags.
- Credentials for a live check are in 1Password (item **PhotoPrism**, user `panbotka`): `POST /api/v1/session` with username+password returns a Bearer `access_token`. Do not commit it; pass it via `KUKATKO_IMPORT_PHOTOPRISM_TOKEN`.
- Docs: `docs/OPERATIONS.md` (the CLI flags) and `docs/PACKAGES.md` (the ppimport entry).
