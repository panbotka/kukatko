# Pick photo location from a map + accept multiple coordinate notations

When editing a photo's metadata (`web/src/components/photo/MetadataPanel.tsx`, editor/admin), latitude/longitude are plain text inputs. Make geotagging visual and forgiving: show a map, let the user pick the position by clicking/dragging, and accept coordinates in several notations like mapy.cz does.

## Requirements

- In the metadata editor, add an **interactive Leaflet map** (over the existing mapy.com **backend tile proxy** — never embed the API key client-side) with a **draggable marker** at the current position.
- **Clicking the map** or **dragging the marker** sets the coordinates; the lat/lng that get saved via the existing `updatePhoto` PATCH flow update accordingly.
- Keep a **text field for coordinates** that accepts multiple common notations (mapy.cz-style), parsed tolerantly:
  - Decimal degrees: `49.1234, 16.5678` (comma or space separated, optional +/-).
  - Degrees-minutes-seconds: `49°7'24.2"N 16°34'12.5"E`.
  - Degrees-decimal-minutes: `49°7.4'N, 16°34.2'E`.
  - Tolerant of extra spaces and N/S/E/W hemisphere suffixes (or signed values).
- Two-way sync: parsing the text moves the marker/map; moving the marker rewrites the text in canonical **decimal degrees**.
- **Invalid** coordinate text shows an inline validation message and blocks saving (do not PATCH garbage).
- Keep a **"clear location"** action (removes coordinates), matching current behavior.
- If the photo has no coordinates yet, open the map at a sensible default (e.g. Czech Republic) and let the user place the marker.
- Large, touch-friendly controls: tap to place, drag to fine-tune, works on mobile.
- Reuse `components/map/LeafletMap` and `services/map` (tile proxy, reverse geocode) rather than adding a new map stack.

## Implementation Notes

- The coordinate parser must be a **pure, unit-tested helper** (e.g. in `web/src/lib/`), returning `{lat, lng}` or a parse error, covering all three notations above and hemisphere signs/suffixes.
- The detail page already has a display-only GPS panel (`PhotoLocation`); this task is about the EDIT experience. Keep them consistent.
