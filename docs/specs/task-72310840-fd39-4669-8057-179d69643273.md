# Trash: admin "delete items older than X days"

Add an admin-only control on the Trash page that permanently deletes archived photos older than a chosen number of days, with the day count entered by the admin each time.

## Background
The Trash page (`web/src/pages/TrashPage.tsx`, route `/trash`) lists archived photos and already has admin-only permanent-delete controls: "Empty trash" (deletes everything), per-card "Delete forever", and bulk "Delete selected". Those call `emptyTrash()` / `purgePhoto()` in `web/src/services/photos.ts` and go through a confirmation `Modal`. Non-admin (editor) users see the page and can restore, but do not see the permanent-delete controls (gated by an `isAdmin` check). There is also a "permanent deletion disabled" server state surfaced via `trash.unavailable`.

A backend endpoint deletes archived photos older than N days: **`POST /api/v1/trash/purge-older?days=<N>&confirm=true`**, `RequireAdmin`, returns a summary with the number of photos purged. `days` must be an integer ≥ 0; `days = 0` purges everything currently in trash. If the exact path/param differs, use what the backend actually exposes — check `internal/photoapi`.

## Requirements
- On the Trash page, add an **admin-only** control (gated by the same `isAdmin` check as the existing "Empty trash" button) to purge items older than a chosen age. Place it near the existing "Empty trash" action in the page toolbar/header.
- The control includes a **number input for days, defaulting to 180**. The admin can change it each time — it is an ad-hoc value, NOT persisted to config. Constrain the input to integers ≥ 0.
- A button (e.g. "Delete older than…") triggers a **confirmation modal**, reusing the existing permanent-delete confirmation `Modal` pattern on this page. The confirmation text must state in words that this permanently deletes all trashed photos older than the entered number of days and cannot be undone — and must include the concrete number the admin typed.
- On confirm, call the backend purge-older endpoint with the entered day count and `confirm=true`. Add a service function in `web/src/services/photos.ts` (e.g. `purgeTrashOlderThan(days)`) following the pattern of the existing `emptyTrash()` / `purgePhoto()`.
- On success: show a toast with the number of photos deleted (from the response) and refresh the trash listing so purged items disappear. On error: show an error toast, matching how the existing "Empty trash" action reports results.
- Respect the existing "permanent deletion disabled" state (`trash.unavailable`): if permanent deletion is disabled server-side, this control is disabled/hidden just like the other permanent-delete controls.
- i18n: add keys in both `web/src/i18n/locales/cs/common.json` and `.../en/common.json` under the existing `trash.*` group. Suggested — English: label "Delete older than", input suffix "days", modal title/body, success "Deleted {{count}} photos". Czech: "Smazat starší než", "dní", "Smazáno {{count}} fotek". Match the pluralization style already used in the file.
