# Slideshow estimate placement

The slideshow duration estimate is currently shown twice: on the start screen before the show begins, and in the caption area during the show. It belongs in exactly one place — next to the speed control, while the show runs.

## Requirements

- The start screen no longer shows any duration estimate before the show begins.
- During the show, the estimated remaining time is displayed immediately next to the speed setting in the slideshow settings area.
- The estimate updates as the show advances and reflects the currently selected speed. Changing the speed updates the estimate at once, without waiting for the next slide.
- The estimate disappears when the show ends.
- The caption area no longer carries the remaining-time text.
- Strings are translated in `cs` and `en`; translation keys that become unused are removed.

## Edge cases

- A paused show keeps showing the estimate, frozen at its current value.
- A show whose photo list is still paging in shows an estimate for what is loaded rather than flickering between values on every page.

## Tests

- Component tests: no estimate is rendered on the start screen, the estimate is rendered beside the speed control during a show, and it is recomputed when the interval changes.
