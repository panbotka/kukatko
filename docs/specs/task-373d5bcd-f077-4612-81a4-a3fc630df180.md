# Slideshow Duration Estimate

Before starting a slideshow the user has no idea how long it will run. A 400-photo album at 5 seconds a slide is over half an hour.

## Requirements

- Everywhere a slideshow can be started — album detail, label detail, the library, and search results — show the photo count and the estimated duration next to the start button, for example "40 fotek, asi 3 min 20 s".
- The estimate is the photo count multiplied by the currently persisted interval setting. It recomputes when the interval changes.
- Inside the player, show progress and remaining time, for example "snímek 7 ze 40, zbývá 2 min 45 s".
- Format compactly, through i18next in both Czech and English: plain seconds below a minute, minutes and seconds above it, and hours when the show runs that long. The string must always fit on one line next to the button.
- Use the photo count the page already has. Do not issue an extra count query and do not block the render on it; if a count is not yet known, omit the estimate rather than showing a placeholder number.

## Verification

`make check` must pass. Add unit tests for the duration formatter covering seconds, minutes with seconds, and hours, and for the remaining-time calculation mid-show. Update `docs/FRONTEND.md`.
