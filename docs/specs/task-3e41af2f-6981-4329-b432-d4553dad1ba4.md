# Make photo multi-select usable on a touch device

This is the most severe mobile bug in the app: on a phone the user cannot start a
photo selection in the main Library at all, and on the other grids the selection
mode gives no visible affordance until the first blind tap. The root cause is that
the per-tile select checkbox is only revealed on `:hover`, which never happens on a
touch screen.

## Current behavior (root cause)

- Each photo tile's round select checkbox is `web/src/components/library/PhotoTile.tsx`
  (`.kk-tile__check`), styled in `web/src/styles/tokens.css` (around lines 598-631):
  it is `opacity: 0` and only becomes visible via `.kk-tile:hover`, `:focus-within`,
  or the `.kk-tile--checks` class (which is applied only once at least one photo is
  already selected).
- The Library (`web/src/pages/LibraryPage.tsx`) runs the grid in "hover-select"
  mode and renders NO "Vybrat" (Select) button. So on a phone the ONLY way to make
  the first selection — the hover-revealed checkbox — is unreachable, and tapping a
  tile just opens the photo detail. The entire batch toolbar is therefore
  unreachable on touch in the app's home grid.
- The other selection grids (Album, Label, Favorites, Search, Subject, Places,
  Trash) have an explicit "Vybrat" button (`web/src/components/organize/SelectionStart.tsx`)
  that enables selection mode and makes the whole tile a tap-to-toggle target — so
  selection works there — BUT with zero photos selected no checkmark is shown yet
  (no hover to reveal it), so the grid gives no hint it is now a selection surface
  until the user taps once.

## Requirements

- On coarse-pointer / touch devices (`@media (pointer: coarse)` or `(hover: none)`),
  the tile select checkbox (`.kk-tile__check`) must be visible AT REST — not hidden
  behind hover — whenever the grid is a selection surface, so a touch user can both
  see and tap it. This must fix the Library (hover-select mode) so a phone user can
  start a selection there, AND make the affordance visible in the explicit
  selection mode on the other grids before the first pick.
- The checkbox must have a comfortable touch hit area (~44px) on coarse pointers,
  even if the visual circle stays small (use padding / a larger invisible hit box).
- Do not change desktop behavior: on fine-pointer devices the checkbox should keep
  its current hover-reveal behavior.
- Selection must remain writer-only exactly as today; do not expose it to viewers.
- Keep the change robust to how each page configures selection — it must work
  whether the grid is in always-available (hover-select) mode or explicit
  ("Vybrat") mode, since different pages use different modes and that wiring may
  change independently.

## Implementation notes

The cleanest fix is almost certainly CSS in `tokens.css`: add a `@media (hover: none)`
/ `(pointer: coarse)` rule that sets `.kk-tile__check { opacity: 1 }` (and enlarges
its hit area) whenever the tile is selectable, mirroring how `.kk-cover-btn` /
`SubjectPhotoTile`'s cover disc already stay visible on touch (`tokens.css` around
line 746). Confirm the exact class/state that marks a tile as "selectable" so the
checkbox only shows when selection is actually possible. Optionally also consider a
long-press-to-select gesture, but the always-visible checkbox is the required fix.
