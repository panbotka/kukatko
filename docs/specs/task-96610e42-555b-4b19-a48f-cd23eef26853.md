# Give albums, tags and people distinct colours

When filtering the library by an album or a tag/label, each active filter shows as a pill chip —
but every chip is the same Superhero **primary orange** (`text-bg-primary`, `#df691a`), so albums
and tags are indistinguishable ("it just adds orange boxes"). Introduce a consistent colour
language that tells albums, tags, and people apart at a glance, and apply it to the active-filter
chips at minimum.

## Current state (for context)

- Active-filter chips render in `web/src/components/library/FilterBar.tsx` as
  `badge rounded-pill text-bg-primary` with a close button; the chip objects are built in
  `web/src/components/library/filterChips.ts`, where album chips (`album:` key) and label chips
  (`label:` key) are styled identically and differ only by their text prefix and key.
- A separate `.kk-chip` style exists in `web/src/styles/tokens.css` used by the organize UI.

## Requirements

- **Define one small, reusable colour convention** for the three entity kinds — album, tag
  (label), person — as a few named tokens/classes (e.g. a distinct hue per kind) that live in the
  shared styles (`tokens.css`/`app.css`), so the mapping is defined once and reused. Pick three
  hues that are clearly distinguishable from each other and from the destructive/close affordance,
  and that read well on the dark Superhero theme. Albums, tags and people each get their own hue;
  the orange primary should no longer be used to mean "album or tag".
- **Apply the convention to the library active-filter chips.** An album chip and a tag chip are
  immediately distinguishable by colour. Keep the existing chip behaviour (label text, count,
  removable close button, tap-target sizing).
- **Apply the same convention wherever these entities appear as chips** so the colour language is
  consistent, not one-off: the photo-detail organize panel chips
  (`web/src/components/photo/OrganizePanel.tsx`) for albums, tags and people, and any grid/tile
  badges that mark album/tag/person. Do not restyle unrelated chips (e.g. flags, ratings).
- Keep accessibility: colour is an aid, not the only signal — chips keep their text label and,
  where helpful, a small leading icon (album / tag / person), so the distinction survives for
  colour-blind users. Maintain sufficient contrast.

## Verification

`make check` must pass. Add/extend tests asserting that an album filter chip and a tag filter chip
carry different colour classes/variants, and that the organize-panel chips use the same convention.
Verify the colours are actually distinct in the running UI (a static style harness against the real
Superhero + app CSS is acceptable). Update `docs/FRONTEND.md` with the colour convention.
