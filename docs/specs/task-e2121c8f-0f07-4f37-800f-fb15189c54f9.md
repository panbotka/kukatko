# Rework the per-user photo flag into a neutral 3-state personal mark

Today each photo has a per-user "pick/reject" flag (Lightroom-style culling) shown on the photo detail via `FlagControl`, plus a separate per-user "favorite" (heart) and a per-user star rating. Reframe the flag as a neutral **personal marking** with three mutually-exclusive states shown as icons: **👁 eye**, **👍 thumbs-up**, **👎 thumbs-down** (plus "none"). The favorite (heart) and star rating stay exactly as they are — this task only changes the flag. The per-user model stays the same (each user sees only their own marks, like favorites).

## Semantics (keep it simple)
- The flag stays a **single, mutually-exclusive** value per user per photo: one of `none` / `pick` / `reject` today, extended with a third state for the eye. Clicking the currently-active icon clears it back to `none` (the existing "clear" affordance).
- Icon ↔ stored-value mapping (KEEP the existing stored strings to avoid rewriting existing rows — this is additive):
  - 👍 thumbs-up ↔ stored value `pick`
  - 👎 thumbs-down ↔ stored value `reject`
  - 👁 eye ↔ **new** stored value `eye`
- Do NOT rename `pick`/`reject` in the database or API — only add the third value. The stored strings are internal; users see the icons + neutral labels.
