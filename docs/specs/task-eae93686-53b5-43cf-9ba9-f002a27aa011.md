# M2 — Search UI + similar photos

Add the search experience (text box with results grid) and a "similar photos" strip usable from
the detail route.

## Context
Read `docs/ARCHITECTURE.md` §6, §13. Backend: `GET /api/v1/search?q=&mode=` (fulltext/semantic/
hybrid, with degraded-mode flag) and `GET /api/v1/photos/{uid}/similar`. Reuse the library grid
components and the urlState convention. react-bootstrap (Superhero), i18n, responsive.

## Requirements
- **Search page**: a prominent search input (in the navbar and/or a dedicated route). Query and
  mode are stored in the URL (`?q=...&mode=hybrid`) so Back works and URLs are shareable.
  Results render in the same virtualized grid as the library, with filters/sort applicable.
- Mode selector (hybrid default; fulltext; semantic). If the API reports **degraded mode**
  (sidecar offline), show a non-blocking notice that semantic search is temporarily unavailable.
- Empty/loading/error states; debounce input; i18n (cs/en).
- **Similar photos**: a reusable component (used later by the detail page) that fetches
  `/photos/{uid}/similar` and shows a horizontally scrollable strip of similar thumbnails linking
  to their detail.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: query/mode reflected in URL and refetch; degraded-mode notice; debounce;
  empty/error states; similar-photos component renders and links correctly. Mock the API.
- Typed components; all text via i18n.
