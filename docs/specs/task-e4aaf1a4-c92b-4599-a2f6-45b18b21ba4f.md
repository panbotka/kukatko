# M0 — Frontend scaffold (React + react-bootstrap Superhero) + embed

Create the React/TypeScript frontend, themed with Bootswatch Superhero, embedded into the Go
binary, with i18n (Czech default) and frontend lint/test wired into `make`.

## Context
Read `docs/ARCHITECTURE.md` §4 (frontend), §13 (frontend), §19.3 (frontend tests). The frontend
is built with Vite and embedded into the Go binary via `//go:embed` and served with an SPA
fallback (non-asset paths → `index.html`, asset paths cached). This mirrors photo-sorter
(`internal/web/static/embed.go`, `serveSPA`).

## Requirements
- `web/` directory: Vite + React 19 + TypeScript. UI: **react-bootstrap** + **Bootswatch
  Superhero** theme (dark). Routing: `react-router-dom`.
- **i18n** via `i18next` + `react-i18next`: Czech (`cs`, default) and English (`en`) resource
  files; a base set of UI strings; language detection with persisted choice.
- Base app shell: top navbar (app name, nav placeholders, language switcher), main content area,
  responsive (mobile/tablet) using Bootstrap grid. A simple Home page hitting `GET /healthz`
  to prove API connectivity.
- Vite build outputs to a directory embedded by Go (e.g. `internal/web/static/dist`). Add the
  Go `//go:embed` glue + SPA fallback handler in `internal/web` and mount it in the chi server.
- Update root `Makefile` `build` target to run `npm ci && npm run build` in `web/` before
  `go build`. Add `make fmt`/`make lint`/`make test` to also run frontend **ESLint** (strict) +
  **Prettier** check and **Vitest** unit tests.
- **ESLint + Prettier** config; **Vitest + React Testing Library** set up with at least one
  component test (e.g. language switcher toggles language) and one test for the API health call.

## Quality gate (mandatory)
- `make check` MUST pass (Go side unaffected; frontend lint/test must pass). Do not break the
  embedded-serve flow: `kukatko serve` must serve the built SPA and `/healthz`.
- Keep components small and typed. No `any` without justification.
