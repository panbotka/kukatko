# Design Layer

The UI works but reads as dated: heavy borders, cramped spacing, inconsistent type sizes, abrupt state changes, and bare empty states. Modernize the look without changing the mandated stack.

## Hard constraint

react-bootstrap + Bootswatch Superhero stay. Do NOT replace the theme, do NOT introduce Tailwind or another CSS framework — this is a binding rule in CLAUDE.md. Build a layer on top of Superhero instead.

## Requirements

- Introduce a token layer of CSS custom properties for the spacing scale, border radii, elevation shadows, and transition durations. Define it in one stylesheet imported once at the app root.
- Establish a typography scale: page titles, section titles, and body text each get exactly one definition. Components must stop setting ad-hoc font sizes.
- Cards and tiles lose their heavy borders in favour of subtle elevation, with a hover state that lifts or highlights. This applies at minimum to album tiles, label tiles, people tiles, and photo grid cells.
- Add a reusable EmptyState component: icon, short title, one-line hint, optional action button. Use it to replace bare texts such as "Bez štítků." and "Bez náhledu" wherever they represent an empty collection.
- Keyboard focus states must stay clearly visible. Do not remove focus outlines.
- Hover, active, and appear transitions use the token durations. Honour `prefers-reduced-motion` and disable non-essential animation when it is set.
- Every new user-facing string goes through i18next with both Czech and English translations. Czech is the default.

## Verification

`make check` must pass (ESLint strict, Prettier, Vitest, Go). Add a unit test for the EmptyState component. Record the token layer and the EmptyState component in `docs/FRONTEND.md`.
