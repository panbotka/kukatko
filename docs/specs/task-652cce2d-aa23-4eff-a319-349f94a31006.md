# M5 — Import/migration admin UI

Add an admin UI to trigger PhotoPrism import and photo-sorter migration, watch progress, and see
run history.

## Context
Read `docs/ARCHITECTURE.md` §9, §10. Backend: `POST /api/v1/import/photoprism`,
`POST /api/v1/import/photosorter`, `import_runs` history (add a `GET /api/v1/import/runs` if not
present — small addition is acceptable here), and the jobs stats API. react-bootstrap (Superhero),
i18n, admin-only.

## Requirements
- Admin-only **Import page** with two sections (PhotoPrism, photo-sorter): a button to start a run,
  showing whether a run is currently in progress, with live **progress** (poll run status + jobs
  stats; show imported/updated/skipped/failed counts).
- **Run history** table from `import_runs`: source, started/finished, status, counts, last error.
- Clear messaging that PhotoPrism stays primary and import is incremental/repeatable; warn before
  starting a large first run.
- Loading/empty/error states; i18n (cs/en); gated to admin role.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: start-import calls the API and reflects in-progress; progress/counts render from
  polled status; history table renders; non-admins cannot access. Mock API.
- Typed components; all text via i18n.
