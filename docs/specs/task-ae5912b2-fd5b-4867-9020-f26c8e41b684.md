# Fix: incremental PhotoPrism re-import overwrites local user edits

A photo already imported from PhotoPrism, then edited in Kukátko, has its edits
silently reverted the next time PhotoPrism re-lists it — including flipping a
user-hidden photo back to public.

## Root cause (verified)
- `internal/ppimport/metadata.go:129-177` (`metadataUpdate`) and
  `internal/ppimport/photos.go:128-137` (`updateExisting`) apply a full-row overwrite
  via `internal/photos/store.go:267-287` (`UpdateMetadata`).
- `metadataUpdate` sets `TakenAt`/`Lat`/`Lng` whenever PhotoPrism holds a value
  (`if !pp.TakenAt.IsZero()`, `if pp.Lat != 0 || pp.Lng != 0`), sets `Private:
  pp.Private` UNCONDITIONALLY, and `Title: firstNonEmpty(pp.Title, existing.Title)`.
- The re-import trigger is PhotoPrism's `UpdatedAt` bumping (re-index, a label change,
  even a view) — entirely outside the user's control.

## Failure scenario
User corrects a wrong capture time / clears or fixes a GPS point / marks a photo
private in Kukátko. PhotoPrism later bumps that photo's `UpdatedAt`; the incremental
run re-lists it and `updateExisting` runs: the corrected time reverts to PhotoPrism's,
the edited GPS is overwritten, a retitle is clobbered, and — worst — a photo the user
hid becomes PUBLIC again.

## Requirements
- Do not let a re-import clobber locally-edited source-owned fields. Introduce
  per-field local-edit provenance, or a photo-level "user edited since import" marker,
  and skip the source overwrite of `taken_at` / `lat` / `lng` / `private` / `title`
  for fields the user has edited.
- At an absolute minimum, NEVER revert the `private` flag from an import (a hidden
  photo becoming public is a privacy regression).
- A genuinely new PhotoPrism value for a field the user has NOT touched may still flow
  in (preserve the incremental "fill gaps / follow upstream" behavior for untouched
  fields).
- Keep the import idempotent and incremental.

## Testing
- Integration tests: edit each field locally, run a re-import that re-lists the photo,
  assert the local edit survives (especially `private` staying hidden), while an
  untouched field still follows PhotoPrism. `make check` must pass.
- Note the precedence rule in `docs/MIGRATION_AUDIT.md`.
