# M2 — Embeddings & faces schema

Add the pgvector tables and HNSW indexes that store image and face embeddings directly in the
database.

## Context
Read `docs/ARCHITECTURE.md` §5 (data model), §6 (vector search). pgvector 0.8.1 is enabled.
Use **`halfvec` (float16)** to halve index memory (important on the Pi) with negligible recall
loss on normalized embeddings. Image embeddings are 768-dim (CLIP), face embeddings 512-dim
(InsightFace). Metric: cosine.

## Requirements
- Migration creating:
  - `embeddings`: photo_uid VARCHAR(32) PK (FK photos ON DELETE CASCADE),
    `embedding halfvec(768)` NOT NULL, model, pretrained, dim INT, created_at. HNSW index
    `USING hnsw (embedding halfvec_cosine_ops) WITH (m=16, ef_construction=200)`.
  - `faces`: id BIGSERIAL PK, photo_uid (FK photos ON DELETE CASCADE), face_index INT,
    `embedding halfvec(512)` NOT NULL, bbox DOUBLE PRECISION[4] (normalized [x,y,w,h] 0..1),
    det_score DOUBLE PRECISION, model, dim INT, created_at; cached cols marker_uid, subject_uid,
    subject_name, photo_width, photo_height, orientation; UNIQUE(photo_uid, face_index). HNSW
    index on embedding (halfvec_cosine_ops, m=16, ef_construction=200).
  - Ensure FK `ON DELETE CASCADE` (fixes a photo-sorter gap where embeddings/faces lacked FKs).
- Go repositories (`internal/embedding` storage or a new `internal/vectors`):
  Save/GetEmbedding, FindSimilar(embedding, limit, maxDistance) using `embedding <=> $vec`,
  Save/List/DeleteFaces, FindSimilarFaces. Set `SET LOCAL hnsw.ef_search = 100` within
  read-only transactions for queries.
- Provide a helper to convert `[]float32` ↔ `halfvec` for pgx/pgvector-go.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): insert embeddings/faces; cosine similarity ordering is correct
  (nearest first); maxDistance threshold filters; cascade delete removes embeddings/faces when a
  photo is deleted; unique(photo_uid, face_index) enforced.
