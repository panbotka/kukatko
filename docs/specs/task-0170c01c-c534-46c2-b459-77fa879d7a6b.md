# M1 — Thumbnailer

Generate and cache derived images (thumbnails/previews) on the Pi, including HEIC/RAW handling,
keeping the binary CGO-free.

## Context
Read `docs/ARCHITECTURE.md` §4 (image processing), §3.2 (S3). CGO must stay disabled — use
pure-Go for JPEG/PNG/WebP and **shell out** to external tools for HEIC/RAW. Mirrors photo-sorter
(`internal/thumb`, `internal/imgconvert`).

## Requirements
- `internal/thumb` package: a registry of sizes producing `fit` (max-side, keep aspect) and
  `tile` (center-crop square) JPEGs. Provide a sensible set, e.g. fit_720/1280/1920/2560/3840 and
  tile_100/224/500 (qualities ~85–90). Make the set easy to extend.
- Cache layout under `storage.cache_path`: `thumb/<aa>/<bb>/<cc>/<hash>_<size>.jpg`
  (sharded by SHA256 prefix). Regenerable from originals; idempotent (skip if present).
- Decoding: pure-Go `image/jpeg`, `image/png`, `golang.org/x/image/webp` and a resize lib
  (`disintegration/imaging`). Apply **EXIF orientation** automatically.
- **HEIC**: shell out to `heif-convert` to an intermediate JPEG, then resize.
- **RAW** (cr2/cr3/nef/arw/dng/raf/orf/rw2/pef/srw): extract embedded JPEG preview via
  `exiftool -b -PreviewImage` (or `dcraw -e`) rather than full demosaic, then resize. Detect
  missing external tools and return a clear error.
- API: `Generate(ctx, photo, sizes...)`, `GenerateAll(ctx, photo)`, `Path(hash, size)`,
  `Open(hash, size)`.
- Parallelize across sizes/photos with bounded concurrency; decode source once per photo.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests with small embedded test images (JPEG/PNG): correct sizes/aspect for fit/crop,
  orientation applied, cache path sharding, idempotent skip. HEIC/RAW paths: test the
  command-construction/branching logic (skip actual conversion if the external tool is absent).
