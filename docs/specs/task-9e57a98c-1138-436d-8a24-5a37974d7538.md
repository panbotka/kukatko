# Bulk Edit

`web/src/components/organize/BulkEditModal.tsx` exposes add album, remove album, add label, and remove label as four plain `Form.Select` dropdowns — one choice each per apply, with no way to filter the options. Alongside the description, location, private, archive, and favorite fields it reads like a form from 1999, and it will not survive a catalog with hundreds of albums and labels.

## Requirements

- Replace each plain select with a searchable multi-select: type to filter, choose several at once. A single apply can add and remove more than one album and more than one label.
- Group the form into clear sections using the project's design layer for spacing and typography: Organize (albums, labels), Metadata (description), Location, and Flags (private, archive, favorite).
- State plainly how many photos the change will affect, and summarise the pending changes before the user applies them.
- Destructive changes — removing from an album or label, archiving — are visually distinct. When the selection exceeds 50 photos, require an explicit confirmation before applying.
- Preserve the backend contract: the whole batch still goes to `POST /photos/bulk` and is applied in one transaction.
- Keep every field the modal supports today. This is a redesign, not a reduction.
