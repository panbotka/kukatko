# Remove the "private photo" feature

The private flag does nothing: no listing excludes private photos, no role is denied them, thumbnails and originals are served like any other photo. It is merely a flag you can filter by — a lock button in the photo detail header that confuses the user. Remove the feature.

(`internal/mediaurl/mediaurl.go` claims in its package doc that "the private flag and the archive are real security boundaries". That is aspirational, not implemented. Update the doc so it only claims what the code does — the archive part stays.)

## Requirements

- Remove the lock toggle from the photo detail header (`web/src/components/photo/PrivacyToggle.tsx` and its usage).
- Remove the "private" filter from the library filter bar and from the map view, including the `private` field in the view state (`web/src/lib/libraryView.ts`, `web/src/lib/mapView.ts`) and its URL query param. A stale `?private=` in a bookmarked URL must be ignored, not crash the page.
- Remove `private` from the photo update API (`internal/photoapi/update.go`), from the list/timeline filters (`internal/photoapi/params.go`, `internal/photos/store_list.go` `ListParams.Private`), from the map feed params, and `set_private` from bulk edit (`internal/bulkapi`, `internal/bulk`, `internal/ctl/bulk.go`).
- **Keep the `private` column** in `photos` (and in albums/labels/subjects). No migration drops it. The importers (`internal/ppimport`, `internal/psimport`) may keep writing it — do not break the import.
- Remove the now-unused i18n keys in `cs` and `en`.
- `make check` passes: no dead code, no unused imports, no orphaned translation keys, lint clean.

## Implementation Notes

- Grep for `private` / `Private` repo-wide before declaring done — the flag reaches `photoapi`, `bulkapi`, `ctl`, `organize`, `people`, the two importers, and the frontend view-state helpers. A dropped field that another package still writes passes `make check` but breaks at runtime, so run the integration tests too.
- The album/label/subject `private` columns and the saved-search "owner-private" wording are unrelated — leave them alone.
