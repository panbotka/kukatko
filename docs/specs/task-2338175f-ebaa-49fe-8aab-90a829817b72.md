# Explain the Maintenance page in plain language

The admin Maintenance page (`web/src/pages/MaintenancePage.tsx`, route `/maintenance`) shows technical terms with no explanation, so an admin can't tell what they mean or whether action is needed. Example: the background job-queue bar shows "Mrtvé: 402" and it's unclear what "Mrtvé" (dead) means or what to do about it. Add short, plain-language descriptions/help for every metric and action on the page. This is a UI/i18n task — no backend or data-model changes.

## What to explain (add a short human description for each)

### 1. Integrity-scan findings (the scan-result table, `maintenance.findings.*`)
Add a short muted explanation for each of these rows so the admin knows what the count means and whether it's a problem. Use accurate wording derived from the code (`internal/maintenance` scan + the finding definitions in `MaintenancePage.tsx` `FINDING_KEYS`):
- **Chybějící originály** (`missing_originals`): photo exists in the catalog but its original file is gone from disk/storage — real data loss risk, worth attention.
- **Osiřelé soubory na disku** (`orphan_files`): an original file sits in storage but belongs to no catalog photo (never imported); the "Import orphan originals" repair adds them.
- **Chybějící náhledy** (`missing_thumbnails`): photo has no generated thumbnail; grid display may be slow/blank. Fixed by the thumbnails repair.
- **Chybějící embeddingy** (`missing_embeddings`): photo has no AI vector, so similarity/content search won't find it; filled once the AI "box" is online.
- **Chybějící detekce obličejů** (`missing_faces`): face detection hasn't run for the photo, so it won't appear under people; filled once the box is online.
- **Chybějící perceptuální hashe** (`missing_phashes`): photo has no perceptual fingerprint used to find near-duplicates.
Also clarify the scan summary line (`maintenance.scan.summary`: photos / files in DB / originals on disk) with a one-line hint of what "drift" between these numbers indicates.

### 2. Background job-queue states (`maintenance.jobs.*` badges: total / queued / running / failed / dead)
Add a legend/help explaining each state. Verify the exact semantics against `internal/jobs/models.go` + `internal/jobs/store.go` before finalizing wording:
- **Ve frontě (queued)**: waiting to be processed — includes jobs scheduled for a retry and jobs deferred while the AI box is offline (they wait without burning their retry budget).
- **Běží (running)**: currently being processed.
- **Selhalo (failed)**: terminal, non-retryable failures (usually 0 — ordinary failures retry and then go to "dead").
- **Mrtvé (dead)**: the job failed even after exhausting all attempts (default max 5) and was dead-lettered. It will NOT run again on its own — it must be manually requeued (on the System-status page via "Znovu zařadit mrtvé úlohy"). A high count usually means a job type keeps failing, or the AI box was unreachable for a long time.
- Confirm what **Celkem (total)** counts (all jobs vs. non-done) from the `JobStats` backend and describe it accurately.
- Add a one-line intro that this bar is the background queue that runs the repairs/backfills, and that "Mrtvé" jobs can be revived on the System page.

### 3. Repairs (`maintenance.repair.*`)
The repair checkboxes are already fairly descriptive; add a brief hint only where helpful (e.g. clarify that repairs enqueue background jobs and don't run instantly, and that embeddings/faces wait for the box). Keep the existing labels.

## Presentation
- Prefer **inline, discoverable** help over hover-only: a short muted `small`/`text-secondary` description line under (or beside) each finding label and a legend under the job badges. Optionally supplement with an info-icon tooltip (`react-bootstrap` `OverlayTrigger`/`Tooltip` triggered by the `Icon` component's `info-circle` — bootstrap-icons only, per project rules), but the text must be reachable without hover (works on touch).
- Keep it compact and scannable — one clear sentence per item, not a wall of text. Match the existing Bootswatch Superhero styling.
