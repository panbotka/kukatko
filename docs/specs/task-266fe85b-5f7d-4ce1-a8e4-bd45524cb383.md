# Frontend: sorting leaderboard page

Add a leaderboard page that displays the sorting competition standings from `GET /review/leaderboard`, so users can see who has sorted the most. Read `docs/FRONTEND.md` first (routing, services, nav conventions, Superhero theme).

## Requirements

- **Service**: add a `web/src/services/` function calling `GET /review/leaderboard?window=...`, returning the ordered standings (`{ user_uid, display_name, yes_count, no_count, total }[]` + the applied window). Type it; no `any`.
- **Page**: a new page (e.g. `web/src/pages/LeaderboardPage.tsx`) at route `/leaderboard`, rendered inside the standard `Layout` shell.
  - A ranked table/list, highest `total` first, showing rank, display name, total decisions, and the Ano/Ne split. Top 3 get a visual medal/badge (bootstrap-icons only, via the `Icon` component). The current logged-in user's row is highlighted (match on their uid from `useAuth()`); if they aren't on the board, show a subtle "not on the board yet — start sorting" hint linking to `/review`.
  - A window toggle: **All-time / Last 7 days / Today** (drives the `window` query param). Keep the selected window in the URL query params (the "Back always works" convention) so it survives navigation/refresh.
  - Loading skeleton and an empty state ("No decisions yet — be the first to sort", linking to `/review`), consistent with the app's existing empty/skeleton patterns.
  - Long lists: virtualize with `react-virtuoso` per project convention if the board can grow large (a plain list is fine for a small board — use judgement).
- **Nav**: add a nav entry in `web/src/components/Layout.tsx` for the leaderboard, visible to **all authenticated users** (viewer and up — NOT gated behind a role group). Use a suitable bootstrap-icon (e.g. a trophy).
- **Route**: register `/leaderboard` in `web/src/App.tsx` behind `RequireAuth` (any logged-in user).
- **i18n**: all strings via i18next in `web/src/i18n/locales/{cs,en}/common.json`, Czech default (e.g. "Žebříček", "Za celou dobu", "Posledních 7 dní", "Dnes", "Ano", "Ne", "Celkem"), English equivalents.

## Tests

- Vitest for the page: renders standings from a mocked service, highlights the current user, switches windows (updates the query param and refetches), shows the empty state, and renders the top-3 medals.

## Docs (Definition of Done)

Add the page, service, route and nav entry to `docs/FRONTEND.md`. `make check` + dev-server start must pass; commit & push (end with the required Co-Authored-By line).
