# Per-User Ratings & Flags — UI

Let users set a 0–5 star rating and a pick/reject flag on photos, filter and sort by them, and use number-key hotkeys. Backend exists: `PUT/DELETE /api/v1/photos/{uid}/rating` (body `{rating?, flag?}`), list/detail responses include the current user's `rating` and `flag`, the list supports `min_rating`, `flag`, and `sort=rating`, and bulk supports `set_rating`/`set_flag`.

## Requirements
- `web/src/services/photos.ts`: add `ratePhoto(uid, {rating?, flag?})` and `clearRating(uid)`; extend the `Photo` type with `rating: number` and `flag: 'none' | 'pick' | 'reject'`; extend `PhotoListParams` with `min_rating` and `flag`, and add `rating` to the sort union.
- A reusable `RatingStars` component (0–5 clickable stars; clicking the current rating again clears to 0) and a `FlagControl` (pick / reject / clear). Optimistic updates with rollback on error, mirroring `useFavorite`/`FavoriteButton`. Add a `useRating(uid, initialRating, initialFlag)` hook.
- Show stars + flag on `PhotoDetailPage` (in the header/metadata area) and a compact star/flag overlay on `PhotoTile` (like the favorite heart; hidden in selection mode). Reject-flagged photos get a subtle visual treatment (e.g. dimmed tile + a reject badge).
- `FilterBar`: add a minimum-rating selector (any / ≥1 … ≥5), a flag filter (any / picks / rejects), and a `rating` sort option. Persist these in the URL view state (extend `lib/libraryView.ts` and its param mapping).
- Hotkeys: on `PhotoDetailPage` and on the focused tile in the grid, number keys `0`–`5` set the rating, `p` = pick, `r` = reject. Must NOT fire while the user is typing in an input/textarea/contenteditable.
- i18n: add cs + en strings to BOTH locale files (Czech default). No hardcoded UI text.
- Vitest tests: `RatingStars`/`useRating` optimistic update + rollback, hotkey sets rating, and a FilterBar rating filter/sort test.
