# Add a one-photo-per-row library layout, default on mobile

On a phone the library grid is cramped — the smallest tiles are ~140px so a narrow screen shows
~2 columns of tiny thumbnails. Add a **one photo per row** layout (each photo full-width, a single
column) as an additional density, and make it the default on mobile. The multi-column grid stays
available on every screen size.

## Current state (for context)

- The grid is `web/src/components/library/PhotoGrid.tsx` (`VirtuosoGrid`, window scroll). Column
  count comes from `GridDensityControl` (`web/src/components/library/GridDensityControl.tsx`), a
  `− / value / +` stepper, backed by `web/src/lib/gridDensity.ts` where `GRID_COLUMNS_MIN = 2`,
  `GRID_COLUMNS_MAX = 8`, a per-device localStorage key (`kukatko.grid.density`, value `auto` or a
  pinned column count), and `gridTemplateColumns()` builds the CSS `grid-template-columns`. Density
  is deliberately per-device (localStorage), not URL state.
- There is no single-column / list mode today; the minimum is 2 columns.

## Requirements

- **Allow a single-column (one photo per row) layout** by extending the density model so the pinned
  column count can go down to 1 (`GRID_COLUMNS_MIN = 1`). At 1 column each photo spans the full
  content width, keeping the same tile/overlay behaviour (favorite, rating, selection, badges,
  click-through to detail) — it is the existing tile at full width, not a new component.
- **Default to one-per-row on mobile.** When the device has no stored density preference and the
  viewport is narrow (phone width), the initial layout is 1 column. On wider screens the default
  stays as it is today (auto/multi-column). Once the user changes the density, respect their stored
  choice on that device regardless of viewport (keep the existing per-device persistence).
- The `GridDensityControl` stepper reaches 1 and shows a sensible label for it (e.g. "1" / a
  single-column indication); the `−` button is disabled at 1 and `+` at the max.
- Full-width single-column tiles must remain performant via the existing virtualization — do not
  break `VirtuosoGrid` window scrolling or infinite paging.
- No change to filters, sorting, URL state, or the API; this is a layout/density addition only.

## Verification

`make check` must pass. Extend the grid/density tests: `gridTemplateColumns(1)` yields a
single-column template; the density control can step down to 1 and disables `−` there; and, given
no stored preference, a narrow viewport resolves to the one-per-row default while a wide viewport
does not. Update `docs/FRONTEND.md` (and `docs/PERF.md` if it documents grid density).
