# M7 — Photo detail page

Build the rich photo detail view that combines the best of PhotoPrism and photo-sorter.

## Context
Read `docs/ARCHITECTURE.md` §13 (detail), §7, §12. Backend already provides: photo detail +
metadata update, EXIF/GPS, faces (`/photos/{uid}/faces` with bbox+assignment+suggestions),
similar (`/photos/{uid}/similar`), labels/albums, favorites, non-destructive edits
(`photo_edits`), map proxy. Reuse the face overlay + similar-photos components built earlier.
react-bootstrap (Superhero), i18n, responsive/touch.

## Requirements
- Large preview (appropriately sized) with prev/next navigation that respects the originating
  list/filter order; deep-linkable (`/photos/{uid}`), Back returns to the prior view.
- **Metadata panel**: view + edit (editor/admin) title/description/notes/taken_at; show camera/
  lens/EXIF; **GPS mini-map** (Leaflet via the mapy.com proxy) with reverse-geocoded location
  on demand; allow setting/clearing location.
- **Faces**: overlay boxes (reuse component), assign/confirm people with suggestions.
- **Similar photos** strip (reuse component).
- **Albums & labels** of the photo: add/remove inline. **Favorite** toggle (per-user).
- **Non-destructive edits**: crop/rotate/brightness/contrast UI writing to `photo_edits`;
  original preserved; download honors edits (streamed).
- Download original button. All actions role-gated; loading/empty/error; i18n (cs/en).

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: metadata edit calls API; prev/next respects order + Back; face overlay + assign;
  similar strip; album/label add/remove; favorite toggle; edit panel writes photo_edits and
  preview reflects it; viewer sees read-only. Mock API/Leaflet.
- Typed components; all text via i18n.
