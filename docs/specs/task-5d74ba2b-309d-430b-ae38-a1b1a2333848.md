# Library As Homepage

`/` currently renders `web/src/pages/HomePage.tsx`: a welcome heading, a grid of card links, an API health badge, and a build version. The photo library — the single most important thing in the app — is one click away at `/library`. Swap that.

## Requirements

- `/` renders the photo library: the virtualized photo grid with its filters, sorting, and pagination, exactly as `/library` does today.
- `/library` redirects to `/` and preserves the query string, so existing links and any saved search that stored the path keep working. Use a replacing redirect so the back button does not bounce.
- The old card-grid landing page is removed. Its remaining content finds a home: move the API health badge and build version to the account page.
- View state (filters, sorting, search, page) stays in URL query params with History API, so the back button always works. This is a project rule.
- The empty catalog case shows a proper empty state with a call to action pointing at upload, not a blank grid.

## Verification

`make check` must pass. Add a routing test asserting that `/library?year=2024` redirects to `/?year=2024`. Update `docs/FRONTEND.md`.
