# M3 — Outlier detection

Detect likely-misassigned faces for a given person by ranking faces by distance from the person's
embedding centroid.

## Context
Read `docs/ARCHITECTURE.md` §7 (outlier detection). Depends on: faces table, subjects/markers.
Mirror photo-sorter: compute a centroid (element-wise average of a person's face embeddings) and
score each face by cosine distance to it.

## Requirements
- `GET /api/v1/subjects/{uid}/outliers` (editor/admin): load all faces assigned to the subject,
  compute the centroid, score each face by cosine distance to the centroid, return faces ranked
  descending (most suspicious first) with distance and photo/thumb info.
- Helper `computeCentroid([]embedding)` and `cosineDistance` (or reuse existing vector utils).
- From the result, the UI can unassign a wrong face (reuse the assignment API) — no new mutation
  endpoint needed here beyond what exists.
- Handle small sets gracefully (1–2 faces → no meaningful outliers).

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests: centroid + cosine distance correctness; ranking order.
- Integration tests (test DB): a planted outlier face ranks at the top for a subject with several
  consistent faces; small-set edge cases.
