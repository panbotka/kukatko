# M0 — CI and release packaging

Add CI that enforces the quality gate and release packaging that produces an arm64 Debian
package with a systemd unit, mirroring photo-sorter's approach.

## Context
Read `docs/ARCHITECTURE.md` §14 (Build & deploy), §19.5 (CI). Target runtime is **Raspberry Pi
(arm64)**. Build is `CGO_ENABLED=0`. Reference photo-sorter's `.goreleaser.yaml` and
`deb/photo-sorter.service` at `/home/pi/projects/photo-sorter/`.

## Requirements
- **GitHub Actions** workflow (`.github/workflows/ci.yml`): on push/PR, set up Go 1.26 + Node,
  run `make check` and frontend lint/test, and run `make test-integration` against a
  **`pgvector/pgvector:pg17` service container** with `KUKATKO_TEST_DATABASE_URL` pointed at it
  (create extensions in a setup step). Cache Go and npm.
- **goreleaser** (`.goreleaser.yaml`): builds `CGO_ENABLED=0` for `arm64` (and `amd64` for dev),
  injects version/commit via ldflags into `internal/version`. Build frontend before packaging so
  the embedded SPA is current.
- **.deb package**: install binary to `/usr/bin/kukatko`, a **systemd unit**
  (`kukatko.service`: runs `kukatko serve`, `Restart=always`, `EnvironmentFile=/etc/kukatko/
  kukatko.env`, dedicated user), an env-file template as a dpkg conffile (preserved on upgrade),
  and create data dirs (`/var/lib/kukatko/{originals,cache}`) in postinstall. Declare apt
  dependencies: `exiftool`, `libheif-examples`, `dcraw` (or libraw equivalent), `postgresql-client`.
  Do NOT depend on texlive (no photo book).
- A release workflow (tag-triggered) running goreleaser is acceptable but optional; CI gate is
  the priority.

## Quality gate (mandatory)
- `make check` MUST still pass. The CI workflow itself must be valid and run `make check`.
- Keep secrets out of CI logs; the test DB in CI is ephemeral. Do not commit real env files.
