# M3 — Face auto-clustering

Group unassigned faces into clusters of the same person so the user can name a whole cluster at
once — the key UX improvement over photo-sorter's manual per-face naming.

## Context
Read `docs/ARCHITECTURE.md` §7 (auto-clustering). Depends on: faces table + vector search,
subjects/markers, assignment API. Faces are 512-dim cosine embeddings.

## Requirements
- Clustering of currently-unassigned faces by embedding similarity (e.g. greedy/connected-
  components over HNSW nearest neighbors within a cosine-distance threshold, or simple online
  clustering). Tunable threshold + minimum cluster size. Deterministic and re-runnable as photos
  are added (incremental: cluster only new/unassigned faces, optionally merge into existing
  named subjects when close to their centroid).
- API (editor/admin):
  - `GET /api/v1/faces/clusters` → list clusters with representative face(s), size, and example
    photos.
  - `POST /api/v1/faces/clusters/{id}/assign` → assign an entire cluster to a subject
    (find-or-create by name), creating markers + assignments for all faces in the cluster.
  - Allow splitting/removing a face from a cluster before assigning (so a stray face doesn't
    pollute a name).
- Suggest a likely existing subject for a cluster (nearest named centroid).

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): faces with similar embeddings cluster together and dissimilar ones
  don't; assigning a cluster creates markers/assignments for all its faces; incremental
  re-cluster only touches unassigned faces; nearest-named-subject suggestion.
