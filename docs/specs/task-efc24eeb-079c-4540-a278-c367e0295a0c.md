# Hero-first photo grid

Make photos the hero of the library. Today the grid reads like cards on a page; modern photo
apps (Apple/Google Photos, Immich) show a dense, edge-to-edge wall of images with almost no
chrome. Reshape the main library grid accordingly.

## Requirements

- Target the main library timeline/grid (the `react-virtuoso`-virtualized photo grid on the
  library page). **Keep virtualization intact** — do not regress scroll performance on large
  libraries.
- **Remove card chrome:** no per-item borders, no card background, no drop shadow, minimal or
  zero corner radius. Each cell is essentially the image.
- **Dense, edge-to-edge layout:** a small uniform gap (≈2–4px) between tiles, grid runs to the
  content edges. Responsive column count that increases with viewport width; tiles stay square
  (center-cropped thumbnails) to keep rows aligned.
- **Hover affordance:** on hover, a subtle bottom-up gradient scrim reveals lightweight
  metadata (e.g. date) and leaves room for the selection control; the image itself lifts
  subtly (no layout shift). On touch, no hover state is required.
- **Date/section headers** (if the timeline groups by day/month) become quiet, sticky, low-
  contrast labels that don't compete with the images.
- Use the design tokens in `tokens.css`/`app.css` for spacing, radius and the scrim — no
  hardcoded colors.
- Keep view state (filters/sort/search/page) in URL query params so Back still works.

## Verification

- Verify visually via the static CSS harness and/or a throwaway instance driven in a browser;
  confirm smooth scrolling with a large grid.
- `make check` must pass.
