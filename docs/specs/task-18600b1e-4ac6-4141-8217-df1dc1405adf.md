# Autocomplete for "add to album" on the photo detail page

In the photo detail Organize panel (`web/src/components/photo/OrganizePanel.tsx`) the "add to album" control is a `Form.Select` dropdown listing all available albums (`albumChoice` / `availableAlbums`). With many albums this is unusable. Replace it with a type-to-filter autocomplete field.

## Requirements

- Replace the album "add" dropdown with an **autocomplete text input**: as the user types, show a dropdown of matching albums (those the photo is NOT already in), filtered by title **case- and accent-insensitively**.
- Selecting a suggestion adds the photo to that album via the existing `addAlbumPhotos` flow and updates the album chips shown in the panel; the input clears after adding.
- **Keyboard support:** arrow keys move through suggestions, Enter selects, Esc closes the suggestion list. Mouse/touch click also selects. Visible focus/hover states.
- Show a clear **"nothing matches"** state when the query matches no album.
- Do **not** create albums from this field — if the typed text matches no existing album, adding is simply unavailable (album creation stays on the Albums page). 
- Apply the **same autocomplete pattern to the label "add" control** in the same panel, for consistency (labels have the same "too many to scroll" problem).
- Must work smoothly with hundreds of albums/labels.
- Only editors see the add controls (unchanged gating).
- Accessible: associated label, appropriate ARIA roles for the combobox/listbox, and a tap target of at least ~44px.

## Implementation Notes

- Prefer a lightweight approach: a filtered dropdown built on `react-bootstrap` primitives is fine and avoids a new dependency. If you do add a library, prefer `react-bootstrap-typeahead`. Describe-the-behavior; the exact widget is your choice as long as the UX above is met.
- The album/label lists are already fetched in the panel for editors; filter client-side.
