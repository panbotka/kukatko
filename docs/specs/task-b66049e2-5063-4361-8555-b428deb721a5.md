# M2 — Semantic & hybrid search

Add natural-language semantic search over photos (text → CLIP embedding → vector search) and a
hybrid mode that fuses it with full-text results.

## Context
Read `docs/ARCHITECTURE.md` §6.2. Depends on: embeddings sidecar client (`TextEmbedding`, 768),
embeddings table + similarity query, and the full-text search added earlier. The box may be
offline — semantic mode must degrade gracefully (fall back to full-text with a clear flag).

## Requirements
- Extend `GET /api/v1/search` with a `mode` param: `fulltext` (existing), `semantic`, `hybrid`
  (default `hybrid`).
  - **semantic**: embed the query text via the sidecar, vector-search `embeddings` (cosine, HNSW),
    return photos ordered by similarity.
  - **hybrid**: combine full-text and semantic rankings with **Reciprocal Rank Fusion (RRF)**
    (documented constant), de-duplicate, return a unified ranked list.
- If the sidecar is unavailable, `semantic`/`hybrid` gracefully fall back to full-text only and
  the response indicates degraded mode (so the UI can inform the user).
- Respect the standard list filters (date/gps/etc.) and pagination across all modes.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB) with a **fake sidecar**: semantic returns vector-ordered results;
  hybrid RRF fusion ordering is correct and de-duplicated; offline sidecar → graceful full-text
  fallback with degraded flag; filters applied in all modes.
