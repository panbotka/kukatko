# Photo Detail

The photo detail page (`web/src/pages/PhotoDetailPage.tsx`) renders the same photo **twice**: once as the main preview, and again at full size inside the "Obličeje" (faces) section — even when no face was detected, where it prints a second copy of the image above the sentence "Na této fotce nebyl detekován žádný obličej." The right-hand panel meanwhile leads with EXIF (camera, lens, aperture, exposure, focal length, ISO), while albums and labels sit far below.

Model the result on photo-sorter's photo detail: one image, toggleable overlays, details on demand.

## Requirements

- Exactly ONE image of the photo on the page. Faces become an overlay drawn on top of that single image, never a second copy.
- The face overlay is toggled on and off by a visible control. Remember the choice across photos.
- When no faces were detected, show a single unobtrusive line. Never render an image for it.
- Promote what matters. Title, description, labels, albums, and faces are the primary content and are visible without any extra click.
- Demote technical detail. Camera, lens, aperture, exposure, focal length, ISO, file name, and dimensions collapse behind an expander that is closed by default.
- Keep the existing capabilities: previous/next navigation, download original, rating, favorite, archive, and the editor-only edit panel.
- Respect the existing RBAC: viewers see everything read-only.

## Verification

`make check` must pass. Add unit tests asserting that a photo with zero faces renders exactly one `img` element for the photo, and that the technical-detail section is collapsed on first render. Update `docs/FRONTEND.md`.
