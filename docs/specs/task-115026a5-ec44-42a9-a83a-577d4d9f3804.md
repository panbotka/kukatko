# Chronological albums

An album always presents its photos oldest first. Both the sort selector and the manual drag-and-drop ordering are removed from albums.

## Requirements

- Album detail lists photos ascending by taken date, oldest to newest. No sort selector is rendered on that page.
- The drag-and-drop manual reordering of album photos is removed from the user interface.
- The API endpoint that persisted a manual album photo order is removed, together with the album field that stored the chosen ordering, via a new numbered migration in `internal/database/migrations/`.
- Photos with no taken date fall back to their upload date, so the ordering is total and stable rather than arbitrary.
- Other photo lists — library, search, labels, person, places — keep their sort selector and current behaviour untouched.

## Implementation notes

- Removing the stored ordering means updating every read and write of it in the organize package, its API layer and its tests. No stale reference may remain; the build and the strict linter must stay green.
- The album photo listing is served by the shared photos endpoint filtered by album, so the ordering must be enforced where albums are listed rather than by changing the shared endpoint's defaults for everyone.

## Tests

- Integration test asserting the album photo listing returns oldest-first regardless of any sort or order query parameters passed to it.
- Integration test asserting the removed reorder route no longer exists.
- A component test asserting album detail renders no sort selector and no drag handles.
