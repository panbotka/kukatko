# Reusable bulk edit

Bulk metadata editing of photos exists only on the library page. Make it a reusable building block and enable it on album detail and label detail.

## Requirements

- Extract the photo selection mechanism and the bulk-edit trigger currently wired into the library page into a reusable component/hook, so any page rendering a photo list can enable bulk editing with minimal wiring.
- Album detail: entering selection mode offers bulk edit alongside the album-specific actions it already has (set cover, remove from album). Bulk edit is added to those actions, it does not replace them.
- Label detail: gains selection mode and bulk edit. It has no selection mechanism today.
- After a successful bulk edit the visible photo list refreshes and the selection clears.
- Bulk editing is offered only to users who may write. Viewers never see the control.
- The photos submitted are exactly the ones selected, never the whole filtered result set.

## Edge cases

- With zero photos selected the edit control is disabled.
- A failed bulk request keeps the selection intact and surfaces the server's error message.
- Removing photos from an album while a selection is active must not leave stale identifiers selected.

## Tests

- Component tests (Vitest): the control is hidden for a viewer, disabled at zero selection, the selection is cleared after a successful edit and preserved after a failed one, and both album and label pages render the trigger.
