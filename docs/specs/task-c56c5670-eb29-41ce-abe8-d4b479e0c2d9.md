# M1 — Storage layer

Implement on-disk storage of original files with content hashing and a deterministic layout.

## Context
Read `docs/ARCHITECTURE.md` §3.2 (S1 Storage), §5. Originals are stored under
`storage.originals_path` as `YYYY/MM/<filename>` (date from taken_at, fallback to import time).
Content identity is **SHA256** (hex). Cache/derivatives are a separate concern (thumbnailer task).

## Requirements
- `internal/storage` package with an interface and a filesystem implementation:
  - `Store(ctx, src io.Reader, takenAt time.Time, originalName string) (StoredFile, error)`
    streams to disk, computing SHA256 while writing (no full buffering in RAM), returns
    {hash, relative path `YYYY/MM/<name>`, size, mime}. Handle filename collisions safely
    (e.g. suffix) without overwriting different content.
  - `Open(ctx, relPath) (io.ReadCloser, error)`, `Stat`, `Delete`, `AbsPath(relPath)`.
  - MIME detection from content (and extension as hint).
- Be safe under concurrency: two simultaneous writes of identical content must not corrupt each
  other; surface a clear "already exists" signal to callers for dedup decisions.
- Stream throughout (large files); never load whole file into memory.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests (use a temp dir): SHA256 correctness, path layout by date, collision handling,
  read-back equals written bytes, delete, concurrent identical writes. No DB needed.
