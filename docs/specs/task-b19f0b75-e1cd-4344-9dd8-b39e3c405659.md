# M1 — Upload & ingest pipeline

Implement the upload endpoint and ingest pipeline that turns uploaded files into photo records
with thumbnails and queued background work.

## Context
Read `docs/ARCHITECTURE.md` §3.2 (S2 Ingest), §5, §8 (jobs). Depends on storage, thumbnailer,
EXIF, and the photos repository (all present in the codebase). Identity is SHA256; near-duplicate
detection uses pHash/embedding distance (config `duplicate.*`). The persistent job queue may not
exist yet — if it does, enqueue embedding/face jobs; otherwise enqueue thumbnails synchronously
and leave a clearly marked TODO hook for embedding/face enqueue that the jobs task will wire up.

## Requirements
- `POST /api/v1/upload` (auth: editor/admin): accepts multipart with one or more files. For each
  file:
  1. Stream to a temp location while hashing (SHA256).
  2. **Exact-duplicate** check by SHA256 → if exists, return a friendly per-file "duplicate"
     result (HTTP 409 semantics in the per-file status, not a hard 500).
  3. Detect format/mime; extract EXIF/GPS; move original into storage (`YYYY/MM`).
  4. Insert `photos` + primary `photo_files` row; compute pHash/dHash → `photo_phashes`.
  5. Generate thumbnails (thumbnailer).
  6. Enqueue `image_embed` and `face_detect` jobs (if the queue exists).
- Return a structured per-file result list (created uid / duplicate / error) so partial batches
  report cleanly. Stream/limit memory; do not buffer entire files in RAM.
- **Near-duplicate warning** (config-gated): if a very similar pHash exists, include a warning in
  the result (do not block).
- Concurrent uploads of identical content must not corrupt storage or double-insert (handle the
  race; loser gets a clean duplicate result, not a 500).

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): single upload creates photo+file+phash+thumbs; exact duplicate
  returns duplicate result; multi-file batch with mixed outcomes; concurrent identical uploads
  yield one photo + a clean duplicate result. Use small embedded test images.
