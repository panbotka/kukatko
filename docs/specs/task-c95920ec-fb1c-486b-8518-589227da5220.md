# Calm the Chrome Around the Photos

Three places where the app's own furniture crowds out the photographs it exists to show.

## Part 1 — On a phone, roughly 80% of the first screen is chrome

On a 390×844 viewport the library's FIRST PHOTO starts at about y=670. You see one partial thumbnail on the opening screen of a photo app. Stacked above it: hamburger, page title, three buttons (Promítání / Uložit pohled / Vybrat), the search box, the sort select, the tile-density control, the Filtry button, a hint line, then FOUR full-width filter selects stacked vertically (Rok, Album, Štítek, Osoba), then "Počet fotek: 280".

Those four selects are NOT the collapsible panel — this was verified in the browser: on a fresh load the Filtry button reports `aria-expanded="false"` while the row is still visible. It is a permanently rendered row, `.kukatko-filter-facets` in `web/src/components/library/FilterBar.tsx`, a sibling of the collapsible `#library-filter-panel`. On desktop it is one tidy row of four; on a phone it becomes four stacked blocks that bury the photos.

### Requirements

- On narrow viewports the four facets must not sit between the search box and the photos. Fold them into the same progressive-disclosure surface the other filters already use — `FilterBar` already renders an `Offcanvas` when `narrow` and a `Collapse` otherwise.
- Any ACTIVE filter stays visible without opening anything. The existing chips (`components/library/filterChips.ts`) are the right vehicle: the user must never lose track of why they are looking at a filtered set.
- Photos must start as close to the top of a phone screen as the header and search allow. Judge the result by where the first thumbnail actually lands, not by whether the code reads tidily.
- Desktop stays as it is — four across in one row is fine at width.
- Do NOT add search to the navbar. That was considered and explicitly rejected; search lives in the library and at `/search`.

## Part 2 — The timeline rail collides with itself

On the library at desktop width, the orange "čvc 2026" bubble on the right-hand timeline scrubber overlaps the "2026" year label beneath it. It reads as a rendering bug, not a design. Make the hovered/current bubble and the year ticks incapable of overlapping. The scrubber is already hidden below 576px (`app.css` ~line 201).

## Part 3 — The navbar has eleven items and no hierarchy

Knihovna · Alba · Štítky · Rozšířit · Procházet▾ · Třídění · Nahrát · Nástroje▾ · Import · Správa▾ · shortcuts · admin. Everything shouts at the same volume: "Nahrát" — a primary everyday action — is styled exactly like "Import", an admin-only chore, and exactly like "Rozšířit", a power-user tool. A non-technical person cannot tell what they are meant to use.

`docs/UX_AUDIT.md` states the navbar's current structure is shipped and that "any further restructuring is separate". This task IS that separate work.

### Requirements

- Give the navbar a visible hierarchy matching how often a normal person uses each thing. The everyday loop — browse, triage, add photos — should be obvious and immediate; admin and power-user tooling stays present but quieter.
- Do NOT bury things behind so much indirection that a known feature becomes unfindable. The audit's line about the review game applies to the whole navbar: "a game nobody can find is a game nobody plays".
- Respect what is settled: no search in the navbar; nav `title` attributes describe the ACTION, not the noun ("Zobrazit alba", not "Alba"); react-bootstrap's `NavDropdown` is deliberately NOT used because it eats the `title` prop the tooltip needs.
- Role gating must not regress — anything already hidden from non-admins stays hidden.
- Check the result at phone width too, where the navbar collapses to a hamburger.

## Constraints

- All user-facing text via i18n, Czech + English parity (`i18n.test.ts`). No hardcoded strings.
- Icons only from `bootstrap-icons` via the `Icon` component, always with a visible text label; decorative ones `aria-hidden`.
- Use `web/src/styles/tokens.css` — `--kk-space-*`, `--kk-radius-*`, `--kk-shadow-*`, `--kk-duration-*` with `--kk-ease-standard`, and the `.kk-*` semantic classes. Do not redefine Bootstrap `--bs-*` variables. No `fs-*`/`h3`/inline `fontSize` on headings.
- Keep the 44px touch floor under `@media (pointer: coarse)` and never remove focus outlines.
- "Back always works": view state (filters/sorting/search/page) stays in URL query params + History API.
- Safe-area insets on the navbar must survive (`env(safe-area-inset-*)`, `viewport-fit=cover`).
- `docs/UX_AUDIT.md` is STALE — it predates `tokens.css`. Where they disagree, tokens win.
