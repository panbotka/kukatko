# Grid density

Let the user decide how many photos sit side by side in a photo grid. The grid is currently width-driven with a fixed minimum tile size and offers no control.

## Requirements

- A control in the library toolbar / filter bar area sets the number of columns, from a minimum of 2 to a maximum of 8 inclusive.
- The choice persists across reloads in `localStorage`, following the same persistence pattern the slideshow settings already use. It is per-device and is not stored on the server.
- At desktop widths the grid renders exactly the chosen number of columns.
- On narrow viewports the grid may fall back to fewer columns so tiles stay usable, but never renders more than the chosen count.
- The setting applies to every page that renders the photo grid, not only the library.
- Virtualised scrolling keeps working at every column count, including when the count changes while the list is scrolled.

## Edge cases

- A missing, out-of-range or corrupt stored value falls back to the current responsive default instead of throwing.
- Changing the count while photos are selected preserves the selection.

## Tests

- Unit tests for the settings hook: clamps out-of-range values into 2..8, ignores corrupt stored JSON, round-trips a valid value.
- A component test asserting the grid reflects a chosen column count.
