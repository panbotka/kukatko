# M1 — Core photo schema

Add the migrations and Go models/repositories for the central photo tables.

## Context
Read `docs/ARCHITECTURE.md` §5 (data model). Use the existing migration runner and pgx DB layer.
UIDs are app-generated VARCHAR(32); `file_hash` is SHA256 hex; originals live on disk as
`YYYY/MM/<filename>`. Kukatko stores external IDs to support PhotoPrism import and photo-sorter
migration.

## Requirements
- Migration creating:
  - `photos`: uid PK, file_hash UNIQUE (SHA256), file_path, file_name, file_size, file_mime,
    file_width, file_height, file_orientation, taken_at, taken_at_source, title, description,
    notes, lat, lng, altitude, camera_make, camera_model, lens_model, iso, aperture, exposure,
    focal_length, exif JSONB, private, archived_at, uploaded_by (FK users SET NULL), timestamps.
    **External-ID columns**: `photoprism_uid VARCHAR(32)`, `photoprism_file_hash VARCHAR(40)`
    (SHA1), `photosorter_uid VARCHAR(32)`. Indexes: taken_at DESC, archived_at partial,
    GIN on exif; indexes on photoprism_uid and photosorter_uid for dedup lookups.
  - `photo_files`: id BIGSERIAL, photo_uid FK CASCADE, file_path, file_hash, file_size,
    file_mime, is_primary, role CHECK(original|sidecar|edited); unique (photo_uid, file_path);
    partial unique index enforcing one primary per photo.
  - `photo_phashes`: photo_uid PK FK CASCADE, phash BIGINT, dhash BIGINT, created_at.
  - `photo_edits`: photo_uid PK FK CASCADE, crop_x/y/w/h REAL nullable, rotation CHECK(0/90/180/270),
    brightness REAL, contrast REAL, updated_at (all-or-nothing crop CHECK).
- Go `internal/photos` package: typed models + repository with methods used now: Create, GetByUID,
  GetByFileHash, GetByPhotoprismUID, GetByPhotosorterUID, UpdateMetadata, Archive/Unarchive,
  and a List with filtering/sorting/pagination scaffold (full filters in the CRUD task).
- Add FK `ON DELETE CASCADE`/`SET NULL` as documented; do not orphan rows.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): create/read/update/archive a photo, unique file_hash enforced,
  one-primary-file constraint, external-ID lookups, cascade deletes for files/phashes/edits.
