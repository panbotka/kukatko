# Bulk edit on the remaining photo lists

Bulk metadata editing of photos must be available on every page that renders a list of photos. This task covers the person (subject) gallery, the places page, the search results page and the favorites page.

## Requirements

- Each of those four pages gains photo selection and a bulk-edit trigger, behaving identically to the library page: enter selection mode, pick photos, open the bulk edit dialog, apply.
- Prefer reusing whatever shared selection/bulk-edit component exists; if the library page still owns that logic inline, extract it first rather than copying it four times.
- After a successful bulk edit the visible list refreshes and the selection clears.
- Bulk editing is offered only to users who may write. Viewers never see the control.
- The photos submitted are exactly the ones selected, never the whole filtered result set.
- Page-specific actions already present on those pages keep working and are not displaced.

## Edge cases

- With zero photos selected the edit control is disabled.
- A failed bulk request keeps the selection intact and surfaces the server's error message.
- On the favorites page, un-favoriting a photo through a bulk edit removes it from the list without leaving it selected.

## Tests

- Component tests (Vitest) per page: trigger hidden for a viewer, disabled at zero selection, list refreshed and selection cleared after a successful edit.
