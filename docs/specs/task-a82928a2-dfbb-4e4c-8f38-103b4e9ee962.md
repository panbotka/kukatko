# Refined navigation & global search

Make the app's top navigation quieter and more modern, and promote global search to a
first-class, keyboard-driven surface.

## Requirements

- **Slimmer sticky header:** reduce vertical bulk; the header sits over scrolling content with
  a subtle translucent / backdrop-blur background (falling back to a solid token surface where
  blur is unsupported) so the photo wall feels continuous beneath it.
- **Cleaner nav:** consistent spacing, quiet inactive items, one clear active state using the
  accent color; icons via the `Icon` component (bootstrap-icons only). On mobile it collapses
  into a compact menu.
- **Global search as a command surface:** a prominent search entry in the header that opens an
  overlay/palette showing grouped results (photos, people, albums, labels, places) from the
  existing global-search endpoint (`GET /search/global`). Results are grouped with headings and
  are keyboard-navigable (Up/Down to move, Enter to open, Esc to close).
- **Keyboard shortcut:** a shortcut (e.g. `/` or Cmd/Ctrl-K) focuses/opens the search overlay
  from anywhere; the shortcut hint is shown in the field. Do not hijack typing inside inputs.
- Search and its open/closed state should not break Back — keep it out of the way of the
  existing URL-driven view state.
- Use design tokens for surfaces, blur, accent and focus ring. i18n (cs default, en) for all
  new strings.

## Verification

- Drive it in a browser: open search via the shortcut, type a query, confirm grouped results
  render and keyboard navigation + Enter open the right entity.
- `make check` must pass.
