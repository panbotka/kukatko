# M4 — Albums, labels, favorites schema

Add the tables and repositories for albums, labels, and per-user favorites.

## Context
Read `docs/ARCHITECTURE.md` §5. Favorites are **per-user** in Kukatko (replacing photo-sorter's
global `photos.favorite`). Use the existing migration runner + pgx layer.

## Requirements
- Migration creating:
  - `albums`: uid PK, slug UNIQUE, title, description, type CHECK(album|folder|moment|state|month),
    cover_photo_uid (FK photos SET NULL), private, order_by, created_by (FK users SET NULL),
    timestamps.
  - `album_photos`: (album_uid FK CASCADE, photo_uid FK CASCADE) PK, sort_order, added_at.
  - `labels`: uid PK, slug UNIQUE, name, priority, timestamps.
  - `photo_labels`: (photo_uid FK CASCADE, label_uid FK CASCADE) PK, source CHECK(manual|ai|import),
    uncertainty, created_at.
  - `user_favorites`: (user_uid FK CASCADE, photo_uid FK CASCADE) PK, added_at.
- Go repositories: albums (CRUD, add/remove/reorder photos, set cover, list with photo counts),
  labels (CRUD, attach/detach to photos, list with counts), favorites (add/remove, list per user,
  is-favorite check). Slug generation for albums/labels.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): album CRUD + add/remove/reorder + cover set-null on photo delete;
  label attach/detach + counts; favorites add/remove idempotent + per-user isolation; cascade
  deletes when photos/users removed.
