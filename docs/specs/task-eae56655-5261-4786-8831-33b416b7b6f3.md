# Touch gestures on the photo detail (mobile)

On a phone the photo detail is awkward to navigate: prev/next exists only as small on-image `‹`/`›`
buttons and the ←/→ keyboard shortcuts, which phones don't have. There are no touch gestures
anywhere in the frontend today. Add the two gestures people expect from a photo app on mobile:
horizontal swipe to move between photos, and pinch/double-tap zoom in the fullscreen view.

## Current state (for context)

- Photo detail: `web/src/pages/PhotoDetailPage.tsx` renders one full-width image with `‹`/`›`
  overlay buttons and ←/→ keyboard navigation that page to prev/next while preserving the source
  list order (`backHref`/the same navigation the arrows use).
- Fullscreen zoom: `web/src/components/photo/Lightbox.tsx` opens by tapping the still image; it has
  its own prev/next paging but no pinch-zoom.
- No `swipe`/`touch`/`pointer` gesture handling exists in `web/src` yet.

## Requirements

- **Swipe left / right on the photo detail image navigates to next / previous** photo, using the
  exact same navigation the `‹`/`›` buttons and ←/→ keys use (same ordering, same URL/state
  behaviour, same wrap/stop-at-ends semantics). A short horizontal drag with a sensible distance/
  velocity threshold triggers it; a mostly-vertical drag must NOT trigger navigation (so page
  scrolling still works), and the gesture must not fire on interactive overlays (face boxes,
  buttons).
- **Fullscreen lightbox supports pinch-to-zoom and double-tap-to-zoom** on touch devices, with
  pan while zoomed, and swipe to page between photos when not zoomed. Reset zoom when changing
  photo or closing. Keep the existing close (button / backdrop / Esc) and desktop behaviour intact.
- Desktop mouse/keyboard behaviour is unchanged; gestures are additive for touch/pointer input.
  Prefer pointer events or a small, well-maintained gesture util over a heavy dependency; if a
  dependency is added, justify it and keep the bundle lean.
- Gestures feel responsive on a mid-range phone and do not fight native scrolling, the browser's
  back-swipe, or the Leaflet map on the page.

## Verification

`make check` must pass. Add tests simulating touch/pointer sequences: a horizontal swipe on the
detail image invokes next/prev (and a vertical drag does not); double-tap toggles zoom in the
lightbox and changing photo resets it. Where full gesture simulation is impractical in jsdom, unit-
test the gesture-decision helper (direction/threshold → action) directly. Update `docs/FRONTEND.md`.
