# Visual signature — refine the design-token layer

Give Kukátko a deliberate, photo-first visual identity that no longer reads as a stock
Bootswatch Superhero theme. This is the foundation the rest of the UI builds on, so it
must land first and only touch the token / base-style layer — **no component rewrites here.**

## Requirements

- Work in the token layer: `web/src/styles/tokens.css` and `web/src/styles/app.css`
  (the existing `--kk-*` custom-property layer over Superhero). Do not fork Bootswatch.
- **Neutral, near-black surfaces instead of Superhero navy.** A photo app should let images
  be the only saturated thing on screen. Move the app background and card/surface tokens to a
  neutral, slightly-warm dark (near-black → dark neutral gray elevation ramp), keeping
  `data-bs-theme="dark"`. Photos must pop against a calm, desaturated chrome.
- Define a clear **elevation ramp** (at least 3 surface levels: page → surface → raised) via
  background + subtle border, not heavy drop shadows.
- **Typography scale:** a small, consistent modular scale (display / heading / body / caption)
  with sensible line-heights and letter-spacing on headings. Pick one accent color used
  sparingly (links, active nav, primary action) — deliberate, not Superhero's default orange.
- **Spacing & radius rhythm:** consolidate spacing and border-radius into a small set of tokens
  and use them; a single, coherent corner radius.
- **Focus ring:** one visible, accessible focus-visible ring token used everywhere (keyboard
  users must always see focus).
- **Fix the hardcoded-white form controls** — form inputs/selects must derive from the surface
  tokens, not a hardcoded white background, so they read correctly on the dark surfaces.
- Maintain **WCAG AA** contrast for text and interactive elements.

## Verification

- Verify visually with the repo's standalone static CSS harness (real Bootswatch + tokens +
  app.css) driven by agent-browser — no server restart, no auth — before committing.
- `make check` must pass.

## Out of scope

- No changes to component markup/logic. This task only reshapes tokens and base styles so the
  other UI work can consume them.
