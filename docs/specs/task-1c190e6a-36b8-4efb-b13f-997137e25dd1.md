# Per-User Ratings & Flags — HTTP API

Expose per-user photo ratings (0–5) and pick/reject flags over HTTP, annotate photo responses with the current user's rating/flag, and support them in bulk edit. The data layer already exists: a `user_ratings` table, `organize` store methods (`SetRating`/`SetFlag`/`GetRating`/`RatingsAmong`), `photos.ListParams` fields (`RatedBy`/`MinRating`/`Flag` + `rating` sort), and the `photos.Photo` `rating`/`flag` fields.

## Requirements
- In `internal/photoapi` add endpoints (mirror `favorites.go` and its nil-safe interface pattern):
  - `PUT /photos/{uid}/rating` body `{rating?: 0..5, flag?: 'none'|'pick'|'reject'}` — sets rating and/or flag for the current authenticated user (any logged-in user, like favorites). 204 on success, 400 invalid value, 404 missing photo, 503 if the ratings backend is not wired.
  - `DELETE /photos/{uid}/rating` — clears the current user's rating + flag. 204.
- Add a `RatingStore` interface to `photoapi.Config` satisfied by `organize.Store`; when nil, the rating endpoints return 503 and annotation falls back to rating 0 / flag 'none'.
- Annotate list responses (`GET /photos`, `GET /search`, `GET /favorites`) and the detail response (`GET /photos/{uid}`) with the current user's `rating` and `flag`, using `RatingsAmong` (one query per page). Reuse the same annotation seam as `annotateFavorites`.
- In `parseListParams`, parse `min_rating` (int) and `flag` ('pick'/'reject') query params into `photos.ListParams`, and accept `sort=rating`. Invalid → 400.
- In `internal/bulk` + `internal/bulkapi` add operations `set_rating` (0..5) and `set_flag` ('none'|'pick'|'reject'), applied per acting user (like `set_favorite`), inside the existing single-transaction bulk apply with audit logging. Validate values; reject unknown → 400.
- Integration tests for the endpoints (set/clear, response annotation, `min_rating`/`flag` filters, `rating` sort) and the new bulk ops against the test DB.

## Done criteria
- Update `CLAUDE.md` and `README.md` (Photos API + Bulk API sections, new query params, new bulk operations).
- `make check` must pass. Follow the project's strict lint and testing rules.
