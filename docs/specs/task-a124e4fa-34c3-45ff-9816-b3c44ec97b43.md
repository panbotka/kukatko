# M6 — Admin system-status dashboard

Build an admin dashboard that surfaces the operational health of the system in one place.

## Context
Read `docs/ARCHITECTURE.md` §8, §14. Backend has: jobs stats + dead-letter (`/api/v1/jobs/...`),
embeddings sidecar health (offline-aware client), import runs, backup status, metrics. This task
aggregates them into a UI (adding small aggregate endpoints where needed).

## Requirements
- Add a `GET /api/v1/system/status` admin endpoint aggregating: embeddings sidecar reachability
  (online/offline), job queue depth by state/type + dead-letter count, last/next backup run +
  result, last import runs per source, storage usage (originals + cache sizes, free space), DB
  reachability, app version/commit.
- **Admin dashboard UI**: cards/sections showing the above, with quick actions: requeue
  dead-letter jobs, trigger backup, trigger import, trigger maintenance scan (links/buttons to the
  respective flows). Auto-refresh via polling. Clear visual state for "box offline → embeddings
  queued, will resume when online".
- i18n (cs/en), admin-only, responsive.

## Quality gate (mandatory)
- Use the **golang-developer** skill for backend; `make check` MUST pass (incl. frontend lint/test).
- Integration test (test DB): `/system/status` returns correct aggregates (job counts, last
  import/backup, storage). Vitest: dashboard renders each section from mocked status, quick
  actions call the right endpoints, non-admins blocked.
