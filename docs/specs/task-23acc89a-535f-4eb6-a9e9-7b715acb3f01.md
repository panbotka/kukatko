# Timeline Scrubber — UI

Add a draggable date rail beside the library grid that lets users jump to any month quickly. Backend: `GET /api/v1/photos/timeline` returns month buckets `{year, month, count, cumulative}` + `total`, ordered newest-first and honoring the same filters as `GET /photos` (`cumulative` = number of photos before that bucket).

## Requirements
- `web/src/services/photos.ts`: `fetchTimeline(params, signal)` → typed buckets (reuse the same param mapping as the photo list so filters match).
- A `TimelineScrubber` component shown alongside the virtualized `PhotoGrid` on `LibraryPage`. It renders year/month tick labels positioned proportionally to each bucket's `cumulative` over `total`. Month labels via `lib/format` locale-aware formatting.
- Interaction: clicking or dragging a position jumps the grid to the corresponding photo using react-virtuoso `scrollToIndex` with `index = bucket.cumulative`. When jumping ahead of what's loaded, trigger the grid to load enough pages to reach that index (drive the existing infinite-scroll loader so the target page is fetched). As the user scrolls the grid, highlight the current month by deriving it from the virtuoso visible range start vs bucket cumulatives.
- Mobile-friendly: a thin touch-draggable rail; on very small widths it may collapse/hide to avoid crowding the grid (follow the existing responsive approach in `styles/app.css`).
- Refetch buckets when filters change; no buckets → render no scrubber. Loading handling that doesn't shift layout.
- i18n cs + en in BOTH locale files for any labels (Czech default).
- Vitest tests: clicking a bucket calls `scrollToIndex` with the right index; the scrubber reflects the active filters; highlight follows the scroll range.
