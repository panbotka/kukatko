# Redesign the library filter bar (UX + visual)

The library filters (`web/src/components/library/FilterBar.tsx`, used on `LibraryPage` and shared by Favorites, album/label/place detail, and Search) currently render a dense row of small labeled controls (search, sort, archived, private, camera, date-from, date-to, min rating, flag). Users find it cramped and intimidating ("scary to use"). Redesign it to be clear, attractive, and easy — WITHOUT changing what the filters do.

## Requirements

- **Compact default state:** a prominent search field plus a **"Filtry" / Filters** button that shows a badge with the count of active filters. Advanced filters (date range, camera, archived, private, min rating, flag) live in an **expandable panel** (desktop collapse/expander) or **offcanvas** (mobile) so the default view is uncluttered.
- **Active filters as chips:** show each active filter as a removable pill/chip, plus a single **"Zrušit filtry" / Clear all** action. Removing a chip clears that one filter.
- **Larger, touch-friendly controls**, generous spacing, clear labels; group naturally related fields (e.g. date-from + date-to shown together as a range).
- Consistent with the app theme (Bootswatch Superhero). Improve the look of active vs inactive states.
- **No behavior regressions:** every existing filter and the sort selector keep working and stay URL-synced through the existing `viewToParams` / `useUrlState` plumbing and `LibraryView` shape. The `showSearch` / `showSort` props (used to hide query/sort on the Search page) must keep working.
- Because the component is shared, verify it still renders and behaves correctly on: `LibraryPage`, `FavoritesPage`, `AlbumDetailPage`, `LabelDetailPage`, `PlacesPage`, and `SearchPage`.
- Responsive: mobile opens advanced filters in an offcanvas/collapse; desktop uses a clean inline expander. All interactive elements meet the ~44px tap-target guideline.

## UI Details

- The search input should be the visual anchor (largest control).
- Empty state (no active filters) should look calm; active state should make it obvious what is filtering the grid.
