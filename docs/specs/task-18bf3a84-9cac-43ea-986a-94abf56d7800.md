# Security audit — read-only report

Provést bezpečnostní audit celého Kukátka a výsledek zapsat do `docs/SECURITY_AUDIT.md`.
Task **nemění produkční kód** — jen zjišťuje a dokumentuje.

## Requirements

- Vytvoř `docs/SECURITY_AUDIT.md` s datem auditu, revizí commitu a shrnutím na 5 řádků nahoře.
- Každý nález má: **ID** (SEC-001…), **závažnost** (critical / high / medium / low / info),
  **soubor:řádek**, **popis útoku** (konkrétní scénář: kdo, co pošle, co získá) a **návrh opravy**.
  Nález bez konkrétního scénáře zneužití nepatří do reportu — vyhoď ho.
- Nálezy seřaď od nejzávažnějších. Pokud v nějaké oblasti není nic, napiš to explicitně
  ("prověřeno, bez nálezu") — mlčení není důkaz.
- **Nic v repu neměň mimo `docs/SECURITY_AUDIT.md`** (+ případně jeden řádek do tabulky
  "Where to find what" v `CLAUDE.md`, pokud se vejde do `make docs-budget`).
  Žádné opravy kódu, žádné nové balíčky, žádné změny konfigurace.

### Rozsah — všechny čtyři oblasti

1. **Backend Go + DB:** auth/RBAC (admin/editor/viewer), sliding sessions, API tokeny (Bearer),
   SQL injection přes pgx, **IDOR** (cizí saved-searches, alba, fotky, subjekty, per-user favorites),
   path traversal ve `storage`, validace a limity uploadu v `ingest`, rate limiting, audit trail.
2. **Frontend (`web/`):** XSS a `dangerouslySetInnerHTML`, kam se ukládá session/token,
   únik citlivých dat do konzole nebo URL, chybějící escapování v uživatelském obsahu.
3. **Data at rest / secrets:** přístup k originálům (FS i R2), signed URL a jejich expirace,
   hashování hesel (bcrypt cost), `.secrets/`, `MAPY_API_KEY` (nesmí opustit server),
   zálohy na S3, co končí v logách (`internal/obs`) a v audit trailu.
4. **Závislosti a build:** `govulncheck` a `npm audit` (spusť je a výstup shrň),
   shell-out na `exiftool` / `ffmpeg` / `heif-convert` / `dcraw` — **command injection přes
   názvy souborů a EXIF hodnoty**, `CGO_ENABLED=0`.

## Implementation Notes

- Postup: nejdřív `docs/ARCHITECTURE.md` + `docs/API.md` pro mapu vstupních bodů,
  pak grep po handlerech a projdi je od hranice systému dovnitř (každý `/api/v1` endpoint:
  kdo se tam dostane a co si může vyžádat).
- Pro každý potvrzený nález si ověř, že je opravdu dosažitelný z HTTP vrstvy — teoretická
  slabina ve funkci, kterou nikdo nevolá s cizím vstupem, je `info`, ne `high`.
- `make check` musí projít (docs-budget!). `make dev` **nespouštěj** — restartoval by běžící
  instanci a task nemění kód.
- Na závěr commit + push, jak velí `CLAUDE.md`.
