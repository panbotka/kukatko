# Attach And Create Labels From A Photo

On the photo detail, `web/src/components/photo/OrganizePanel.tsx` renders an autocomplete placeholder "Přidat do alba" for albums, but the labels section shows only "Bez štítků." with no control at all. Because the catalog currently holds zero labels (`GET /api/v1/labels` returns an empty list), there is no way to attach a first label to any photo. Adding a label must feel exactly like adding an album.

## Requirements

- The label control mirrors the album control: same placement in the panel, same autocomplete behaviour, same removable chips.
- The control renders even when the catalog contains no labels yet. An empty option list must not hide it.
- Typing a name that does not exist offers to create it — for example "Vytvořit «Dovolená»" — and creating it attaches it to the photo in one action.
- If creation or attachment fails, surface the error and keep the typed text so the user can retry.
- Respect existing RBAC: editors and admins can attach, create, and remove; viewers see chips read-only with no control.
- Reuse the existing services and endpoints rather than inventing new ones, unless creating a label from this flow genuinely needs a new endpoint — in which case document it in `docs/API.md`.

## Verification

`make check` must pass. Add a component test that starts from an empty label list, types a new name, and asserts that the create-and-attach flow calls through and renders the resulting chip. Update `docs/FRONTEND.md`.
