# Language Setting

The cs/en switcher sits permanently in the navbar (`web/src/components/Layout.tsx`), spending prime space on a setting nobody will touch — every user of this instance is Czech.

## Requirements

- Remove the language switcher from the navbar.
- Add a language section to the account page (`web/src/pages/AccountPage.tsx`) offering Czech and English.
- Czech stays the default: for a new user, and whenever no preference has been stored.
- Keep the existing persistence mechanism and the existing i18next wiring; this is a relocation, not a rewrite.
- The switcher component stays reusable and keeps its test; update the test to cover it rendering on the account page.
