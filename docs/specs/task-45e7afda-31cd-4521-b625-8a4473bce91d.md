# M7 — Performance pass

A focused performance pass targeting the Raspberry Pi runtime and large libraries.

## Context
Read `docs/ARCHITECTURE.md` §4, §6, §16 (risks: HNSW memory/build on Pi, pure-Go thumbnail speed).
Runs on ARM with limited RAM; vectors use halfvec + HNSW. This pass measures and tunes; avoid
changing external behavior/contracts.

## Requirements
- **Vector search**: validate HNSW params (`m`, `ef_construction`, query `ef_search=100`, never
  ≥400) and `halfvec` choice on representative data; document index build guidance
  (`maintenance_work_mem`, optionally building the index on a beefier host). Ensure read queries
  set `ef_search` per the design.
- **Thumbnails**: measure pure-Go thumbnail throughput on ARM; if too slow/memory-heavy for large
  images, add an optional `vipsthumbnail` shell-out path (config-gated) while keeping pure-Go
  default and CGO disabled. Ensure bounded concurrency.
- **Queries/pagination**: verify the hot list/search/album queries use proper indexes and
  cursor-based pagination (no large OFFSET scans); add missing indexes via migration.
- **Frontend**: verify grid virtualization, thumbnail lazy-loading, and request batching keep
  large libraries smooth.
- Capture before/after numbers in `docs/` (a short perf notes file) and the commands used.

## Quality gate (mandatory)
- Use the **golang-developer** skill for Go changes. `make check` MUST pass.
- Add benchmarks/tests where meaningful (e.g. thumbnail benchmark, query plan assertions via
  EXPLAIN in an integration test, vector query correctness unchanged after tuning). No regressions
  in existing tests.
