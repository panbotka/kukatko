# Show who uploaded each photo

## Summary
On the photo detail view, show which user uploaded the photo, resolved to a human-readable name (not a raw ID).

## Background (already in place)
- The `photos` table already has an `uploaded_by` column (nullable FK to `users(uid)`), populated at ingest from the authenticated uploader; it is already serialized as a raw UID in photo API payloads. **No migration and no ingest change are needed.**
- The `users` table has `username` and `display_name`; a user can be resolved by UID via the existing auth store (`auth.Store.GetUserByUID`).

## Requirements
- The photo **detail** response exposes the uploader as a small object with the user's display name (falling back to username when display name is empty), alongside the UID â€” not just the raw UID.
- The detail view renders an "Uploaded by" row (i18n label; cs default e.g. "NahrĂˇl/a", en "Uploaded by") showing that name.
- For photos with **no** uploader (null `uploaded_by` â€” e.g. items imported from PhotoPrism / photo-sorter, or legacy rows), show a neutral fallback (e.g. "â€”" or an "imported/unknown" label); do not render a broken/empty row.
- Resolution happens **server-side** and is limited to the **detail** endpoint â€” do not add per-item user lookups to list/grid/search responses (avoid N+1).

## Implementation notes
- Backend: add a minimal user-resolver dependency (an interface with a `GetUserByUID`-style method, satisfied by `auth.Store`) to `internal/photoapi` (`Config`/`API` in `internal/photoapi/http.go`). In `handleDetail`, when `photo.UploadedBy != nil`, resolve the user and attach an `uploader` object to the detail payload struct. Wire the resolver where the API is constructed: `cmd/kukatko/photos.go` (`buildPhotoAPI`).
- Keep the resolver behind an interface so it can be faked in tests (project convention: external deps behind interfaces).
- Frontend: add the field to the photo/detail TypeScript type and API client, and render the row in the detail metadata panel (`web/src/pages/PhotoDetailPage.tsx` / the metadata panel component in `web/src/components/photo/`). Add cs + en i18n keys.

## Tests
