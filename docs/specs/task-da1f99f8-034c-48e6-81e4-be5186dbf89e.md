# Show photo-select checkboxes immediately on all list pages (like the Library)

Make multi-select consistent across every photo-list page. In the Library
(Knihovna) each photo tile already shows a round select checkbox in the top-left
(revealed on hover, pinned once anything is selected), so selecting multiple
photos is immediately available. On the Album detail, Label detail, Favorites,
Search results, and person/subject gallery pages the user must FIRST click a
"Vybrat" (Select) button to enter selection mode before any checkboxes appear.
Unify them: every list should behave like the Library — checkboxes available
immediately, no "Vybrat" button needed.

## Current behavior (root cause)

- Selection is driven by `web/src/hooks/useBulkEdit.ts`, which wraps
  `web/src/hooks/useSelection.ts`. The one difference is the `hoverSelect` option.
- The Library calls `useBulkEdit({ onEdited: reload, hoverSelect: true })`
  (`web/src/pages/LibraryPage.tsx`). Every other list page calls
  `useBulkEdit({ onEdited: reload })` with no `hoverSelect`.
- In `useBulkEdit.ts`, when `hoverSelect` is true the grid selection is always
  defined for a writer; when it is absent the grid selection is `undefined` until
  the user enables selection mode. `web/src/components/library/PhotoGrid.tsx`
  computes `selectable = selection.active || selection.hoverSelect === true`, so
  Library tiles are always selectable while the other pages are not until
  `selection.enable()` is called by the "Vybrat" button
  (`web/src/components/organize/SelectionStart.tsx`).
- Pages that currently gate selection behind "Vybrat":
  `web/src/pages/AlbumDetailPage.tsx`, `web/src/pages/LabelDetailPage.tsx`,
  `web/src/pages/FavoritesPage.tsx`, `web/src/pages/SearchPage.tsx`, and the
  person/subject gallery page (`web/src/pages/SubjectDetailPage.tsx` — confirm the
  exact filename). Grep for `SelectionStart` and for `useBulkEdit(` to find every
  call site.

## Requirements

- On Album detail, Label detail, Favorites, Search results, and the
  person/subject gallery, each photo tile must show the same always-available
  select checkbox as the Library: revealed on hover, pinned once at least one
  photo is selected. Selecting photos must work without any prior click.
- Remove the now-redundant "Vybrat" entry point (`SelectionStart`) from those
  pages. If `SelectionStart` becomes entirely unused across the app afterwards,
  delete the component; if it is still used somewhere, leave it there.
- Only writers (users with write permission) get selection, exactly as today —
  do not expose selection to read-only viewers.
- Keep each page's existing behavior intact otherwise (data loading, empty
  states, virtualization via react-virtuoso, URL-driven view state). This task is
  ONLY about making the select checkboxes appear immediately; it does not change
  which bulk actions are offered.
