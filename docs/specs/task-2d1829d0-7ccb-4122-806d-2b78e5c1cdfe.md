# Select albums and tags when uploading photos

## Summary
On the upload page, let the user choose one or more albums and one or more tags/labels; every photo the upload resolves to is then added to the chosen albums and gets the chosen labels.

## Requirements
- The upload page shows two searchable multi-selects near the drop zone: **Albums** and **Tags/Labels**. Both optional ŌĆö uploading with none selected behaves exactly as today.
- Each selector lists existing albums/labels (fetched from the API) and supports inline creation of a new album/label when the user types a name that doesn't exist (reuse the existing create affordance).
- The selection applies to the whole upload batch (not per-file).
- After all files in the batch finish uploading, every resolved photo is added to the selected albums and gets the selected labels, in as few API calls as possible.
- Photos detected as **duplicates** (already in the library) are also assigned ŌĆö assignment applies to every photo UID the upload resolved to, new or duplicate (the upload response returns a `photo_uid` for duplicates too).
- After uploads complete, show an "assigningŌĆ”" state, then report success, or a clear, **retryable** error if assignment fails (the photos are already uploaded; only album/label assignment failed).
- If nothing is selected, make no assignment call.

## Implementation notes
- Upload page: `web/src/pages/UploadPage.tsx`; queue hook `web/src/hooks/useUploadQueue.ts` already aggregates created photo UIDs (`createdUids`) and exposes an `isComplete` state.
- Reuse the existing searchable multi-select with inline-create: `MultiSelect` in `web/src/components/` (as used by `BulkEditModal`).
- Listing + assignment APIs already exist ŌĆö no backend change needed:
  - List: `GET /api/v1/albums`, `GET /api/v1/labels` (services `fetchAlbums`/`fetchLabels` in `web/src/services/organize.ts`).
  - Assign both at once: `POST /api/v1/photos/bulk` with `{ photo_uids, operations: { add_to_albums, add_labels } }` (service `bulkUpdatePhotos` in `web/src/services/bulk.ts`). Use this single call.
- Ensure the duplicate outcome's `photo_uid` is included in the UID set used for assignment.

## Tests
- Frontend tests (Vitest): selectors render and are optional; after a simulated completed batch the bulk-assign call is made with the collected UIDs and selected album/label IDs; no call is made when nothing is selected. Mock the services; follow existing `web/src` patterns.
