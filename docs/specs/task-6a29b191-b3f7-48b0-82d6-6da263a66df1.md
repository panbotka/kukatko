# M0 — Configuration (YAML + env)

Add a typed configuration layer loaded from a YAML file with environment-variable overrides.

## Context
Read `docs/ARCHITECTURE.md` §14 (Konfigurace). Backend is Go. A PostgreSQL database is already
provisioned; its DSN lives in the gitignored `.secrets/db.env` as `KUKATKO_DATABASE_URL`
(and `KUKATKO_TEST_DATABASE_URL` for tests). Secrets come primarily from env. The mapy.com API
key is `MAPY_API_KEY` in the same file.

## Requirements
- `internal/config` package with a typed `Config` struct and `Load()` using `spf13/viper`
  (YAML file + env override; env wins). Env var prefix `KUKATKO_` mapping to nested keys
  (e.g. `KUKATKO_DATABASE_URL` → `database.url`, `KUKATKO_WEB_PORT` → `web.port`).
- Config keys (with sensible defaults where applicable):
  - `database.url` (required; from `KUKATKO_DATABASE_URL`), `database.max_open_conns` (25),
    `database.max_idle_conns` (5)
  - `storage.originals_path` (default `/var/lib/kukatko/originals`), `storage.cache_path`
    (default `/var/lib/kukatko/cache`)
  - `web.host` (0.0.0.0), `web.port` (8080), `web.session_secret`, `web.allowed_origins`
  - `embedding.url` (default `http://localhost:8000`), `embedding.image_dim` (768),
    `embedding.face_dim` (512)
  - `auth.bootstrap_admin_username`, `auth.bootstrap_admin_password`
  - `maps.mapy_api_key` (from `MAPY_API_KEY`)
  - `backup.s3.{endpoint,region,bucket,access_key,secret_key,path_style}`, `backup.schedule`
  - `trash.retention_days` (30), `duplicate.enabled` (true), `duplicate.phash_max_diff` (8),
    `duplicate.embedding_max_dist` (0.05)
- `Load()` resolves the YAML path from `--config` flag / `KUKATKO_CONFIG` env / default
  `config.yaml`. Validate required fields and return descriptive errors.
- Provide a committed **`config.example.yaml`** documenting every key. Do NOT commit a real
  config; `*.local.yaml`, `.env*`, `.secrets/` are already gitignored.
- Wire `kukatko serve` to load config and use `web.host`/`web.port`.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests for `Load()`: defaults, env override precedence, required-field validation,
  nested key mapping (use `t.Setenv`). No external dependencies in these tests.
