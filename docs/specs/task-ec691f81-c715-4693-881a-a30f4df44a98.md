# M4 — Bulk edit & favorites UI

Add multi-select on the photo grid with a bulk-edit toolbar, and per-user favorite toggles
throughout.

## Context
Read `docs/ARCHITECTURE.md` §1, §13. Backend: `POST /api/v1/photos/bulk` (add/remove album,
add/remove label, set caption/location/private, archive), favorites
`PUT/DELETE /photos/{uid}/favorite`, `is_favorite` in list/detail, `favorite=true` filter.
Reuse the library grid + urlState. react-bootstrap (Superhero), i18n, responsive.

## Requirements
- **Selection mode** in the grid: select multiple photos (click/shift/range; select-all-in-view),
  with a sticky **bulk toolbar** offering: add/remove to album, add/remove label, set/clear
  description, set/clear location, set private, archive. Show per-photo result summary after apply.
- **Favorites**: a heart toggle on grid tiles and detail; a "Favorites" view/filter using
  `favorite=true`. Optimistic updates with rollback on error.
- All actions role-gated (viewer can favorite — personal — but not bulk-edit). Progress + result
  feedback; i18n (cs/en); touch-friendly.

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: multi-select interactions; bulk toolbar calls the bulk API with the right payload
  and renders results; favorite toggle calls API + optimistic update + rollback on failure;
  viewer sees favorites but not bulk-edit. Mock API.
- Typed components; all text via i18n.
