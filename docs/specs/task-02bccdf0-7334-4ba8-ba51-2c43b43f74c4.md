# M2 — Embeddings sidecar client

Implement the HTTP client to the embeddings service (runs on the GPU box), with the exact
contract from photo-sorter, plus health/availability detection so the app degrades gracefully
when the box is offline.

## Context
Read `docs/ARCHITECTURE.md` §6.1 (sidecar contract), §8 (box offline). Base URL from
`embedding.url`. Same contract as photo-sorter (`internal/fingerprint/embedding.go`). The box is
often offline — the client must detect this and let callers (jobs) wait rather than fail hard.

## Requirements
- `internal/embedding` package behind an **interface** (so it can be faked in tests):
  - `ImageEmbedding(ctx, img io.Reader) ([]float32 /*768*/, model, pretrained, error)` →
    `POST {url}/embed/image` multipart field `file`; response `{dim,embedding,model,pretrained}`.
  - `TextEmbedding(ctx, text string) ([]float32 /*768*/, ...)` → `POST {url}/embed/text`
    JSON `{text}`.
  - `FaceEmbeddings(ctx, img io.Reader) ([]Face, model, error)` → `POST {url}/embed/face`
    multipart `file`; response `{faces_count, model, faces:[{face_index,dim(512),embedding,
    bbox:[x1,y1,x2,y2] px, det_score}]}`.
  - `Healthy(ctx) bool` — cheap availability check (HTTP reachability / health route) used to
    decide whether to attempt embedding jobs.
- Sensible timeouts, context cancellation, and **typed errors** distinguishing "service
  unavailable/offline" (retryable, no attempt count burn ideally) from "bad response".
- Validate returned dimensions (768 image/text, 512 face); error on mismatch.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit/contract tests against an `httptest` fake server implementing the three endpoints +
  health: correct multipart/JSON requests, response parsing, dim validation, offline/unavailable
  error classification, timeout handling. No real network/box required.
