# Slideshow Speed

The speed dropdown renders every option identically, as a bare " s" with no number, so the setting appears not to work.

## Root cause (already diagnosed — verify, then fix)

`web/src/components/slideshow/Slideshow.tsx` around line 281 calls the translation with a variable named `count`, while the string `slideshow.speed.seconds` in `web/src/i18n/locales/cs/common.json` and `web/src/i18n/locales/en/common.json` interpolates `{{seconds}}`. The interpolated variable is never supplied, so i18next renders an empty value. The key is used in exactly one place; this is not a duplicate-key problem.

## Requirements

- Pass the value under the name the string interpolates, or rename the placeholder so the two agree. Each option must show its own number.
- The offered intervals become 1, 2, 3, 5, 10, 15, and 30 seconds. The default stays 5 seconds.
- A previously persisted interval that is not in the new set — the current defaults include 7 seconds, stored under the `localStorage` key `kukatko.slideshow.settings` — must snap to the nearest offered value rather than being dropped or rendering as a blank option.
- Changing the speed takes effect from the next slide onward, without restarting the show.

## Verification

`make check` must pass. Add unit tests asserting that every option renders its number, and that a stored 7000 ms value resolves to a valid offered interval.
