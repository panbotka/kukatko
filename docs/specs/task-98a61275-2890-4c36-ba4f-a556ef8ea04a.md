# M1 — EXIF / GPS extraction

Extract photo metadata (capture time, GPS, camera/lens, dimensions, orientation, raw EXIF) for
use during ingest.

## Context
Read `docs/ARCHITECTURE.md` §4 (image processing), §5 (photos columns). Keep CGO disabled. Mirror
photo-sorter's approach: `exiftool` subprocess with a pure-Go fallback.

## Requirements
- `internal/exif` package: `Extract(ctx, path) (Metadata, error)` returning: taken_at (+ source:
  exif/filename/unknown), lat/lng/altitude, camera_make/model, lens_model, iso, aperture,
  exposure, focal_length, width, height, orientation, mime, and the full EXIF as a JSON-able map.
- Primary path: shell out to `exiftool -json` and parse; **fallback** to a pure-Go EXIF reader
  (`rwcarlsen/goexif` or `dsoprea/go-exif`) when exiftool is unavailable.
- GPS: convert EXIF rational coordinates to decimal lat/lng (respect N/S/E/W refs).
- taken_at resolution: EXIF DateTimeOriginal preferred; if absent, attempt filename date parsing;
  else mark source unknown.
- Robust to files without EXIF (e.g. PNG) — return zero values, not errors.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Unit tests with small embedded sample images covering: GPS decimal conversion (incl. hemispheres),
  taken_at + source resolution, missing-EXIF handling, orientation parsing. Test the fallback
  parser path directly (do not require exiftool to be installed for the unit suite).
