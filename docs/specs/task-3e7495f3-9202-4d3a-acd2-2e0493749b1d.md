# Photo detail: consistent empty state for albums and labels

In the "Uspořádání" panel on the photo detail page the two empty states are visually inconsistent: a photo in no album gets a small caption ("V žádném albu"), while a photo with no label gets a large icon-based placeholder.

## Requirements

- Both empty states render the same way: a **small muted caption**, matching the existing album one. No large icon, no oversized placeholder block.
- The label caption keeps its own wording (e.g. "Bez štítku"), translated via i18n in both `cs` and `en` — do not hardcode text.
- Vertical rhythm of the panel stays consistent whether either list is empty or populated (an empty list must not make the panel jump in height relative to the other).
- Touch only the empty states; the populated album/label chips stay as they are.

## Implementation Notes

- The panel is `web/src/components/photo/OrganizePanel.tsx` (renders `photo.albums` and `photo.labels`).
- Reuse the album empty-state markup/classes for labels rather than restyling both.
- Update the component's Vitest coverage for the empty case.
