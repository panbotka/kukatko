# Make the repository's documentation and code English-only

The project rule (see `CLAUDE.md` § Language) is: code, comments, commits, and identifiers in English; only UI copy is bilingual via i18n (Czech default, English). Several developer-facing files are currently written in Czech — most notably `README.md` (fully Czech, ~2000 lines). Translate them to English. This is a low-priority cleanup; do it carefully so nothing functional breaks.

## Priority order (do in this order; commit progress even if you can't finish everything)

1. **`README.md`** — the primary target. Translate the entire file to fluent, natural English. Keep the product name **"Kukátko"** as-is (proper noun). Keep code blocks, commands, env-var names, URLs, and file paths unchanged. Preserve the document structure and Markdown.
2. **`CLAUDE.md`** — translate the few Czech lines (e.g. the `## Package map` entries for `internal/dirimport`, `internal/sidecar`, and any other Czech descriptions) to English. Keep it tight — `make docs-budget` enforces a 300-line limit on this file, so don't let translation add lines; match the existing terse one-line style.
3. **Top-level `docs/*.md`**: `docs/API.md`, `docs/ARCHITECTURE.md`, `docs/FRONTEND.md`, `docs/MCP.md`, `docs/MIGRATION_AUDIT.md`, `docs/OPERATIONS.md`, `docs/PACKAGES.md`, `docs/RESTORE.md`, `docs/SECURITY_AUDIT.md`, `docs/UX_AUDIT.md` — translate any Czech prose/section headings to English.
4. **Go comments and doc-comments** in these non-test files that contain Czech: `cmd/kukatko/importcmd.go`, `internal/expand/expand.go`, `internal/mapy/mapy.go`, `internal/mcpapi/tools_search.go`, `internal/organize/slug.go`, `internal/people/slug.go`, `internal/photoapi/update.go`, `internal/ppimport/scope.go` — translate **comments only**.
5. **Go test files** with Czech **comments** — translate the comments only (there are many, e.g. under `internal/*_test.go`); this is lowest priority within the task.

## MUST NOT change (functional Czech / intentional — leave exactly as-is)

- **`web/src/i18n/locales/cs/**`** — the Czech UI translations. Czech is the default UI language; these are a feature, not a defect. Do not touch the `cs` (or `en`) locale files.
- **Diacritic/transliteration tables and Czech-folding logic** — e.g. the Czech→ASCII maps in `internal/organize/slug.go` and `internal/people/slug.go`. The Czech characters there are functional data (ě→e, š→s, …); translating them would break slug generation. Only translate surrounding comments.
- **Czech string literals that are test inputs or expected values** exercising Czech-language behavior — Czech photo titles/names, search queries testing unaccent/diacritics, slug expectations, Czech place names in `internal/mapy`. These MUST stay Czech or the tests lose their purpose. In test files, translate only the **comments**, never the Czech test data.
- **`docs/specs/task-*.md`** — auto-synced copies of Botka task specs (regenerated by the runner, and they legitimately quote Czech UI strings). Do not translate; consider excluding this directory from your sweep.
- **`docs/superpowers/**`** — historical planning artifacts; skip (out of scope for this cleanup).
- When a doc *quotes actual Czech UI text* to describe the interface (e.g. a button labelled "Ano"/"Ne"/"Nevím", "Žebříček"), keep the quoted Czech string and translate only the surrounding explanatory prose.

## How to verify coverage

- After translating, sweep for remaining Czech-specific characters to confirm nothing developer-facing was missed, excluding the allowed locations, e.g.:
  `grep -rlIE '[ěščřžůňďť]' --include='*.md' --include='*.go' --exclude-dir=node_modules --exclude-dir=dist .` then subtract `web/src/i18n/locales/`, `docs/specs/`, `docs/superpowers/`, the slug transliteration tables, and legitimate Czech test data. (Note: `rg` is not installed on this box — use `grep`.)

## Constraints

- **Do NOT run `prettier --write` on `docs/*.md`** — these docs are hand-wrapped and are not part of the Prettier gate; auto-formatting would rewrap entire files. Edit prose in place and keep the existing wrapping style.
- Keep translations accurate and idiomatic; do not drop information or restructure documents beyond what translation requires.

## Definition of Done

`make check` must pass (it runs `docs-budget` + fmt + lint + typecheck + tests + frontend — translating comments and prose must not break any of these; if a test breaks, you changed a Czech literal you shouldn't have — revert that literal). The dev-server start check must pass. Commit (in English) and push (end the commit message with the required `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>` line).
