# Global Search — UI

Surface cross-entity search (albums, labels, people, photos) in the UI. Backend: `GET /api/v1/search/global?q=` returns `{query, albums, labels, people, photos}` (top-N each). The existing photo fulltext/semantic search on `/search` stays as the main photo results.

## Requirements
- `web/src/services/search.ts` (new) or extend `photos.ts`: `globalSearch(q, signal)` → typed grouped results (`GlobalSearchResult` with album/label/person/photo item types).
- `NavbarSearch`: as the user types (debounced ~250ms), show a quick-results dropdown grouped by type (Albums / Labels / People / Photos), a few items each, with thumbnails/avatars where available. Clicking an item navigates to that entity: album → `/albums/{uid}`, label → `/labels/{uid}`, person → `/people/{uid}`, photo → `/photos/{uid}`. Pressing Enter still navigates to the full `/search?q=…` page. The dropdown is keyboard-navigable and closes on blur/Escape.
- `SearchPage`: above the photo results grid, render compact sections for matching Albums, Labels, and People (cards/chips linking to each) when the global search returns any — so a text query also surfaces non-photo entities. The existing photo search results/grid remain below.
- Empty/loading states for the dropdown; no matches → a subtle "Nic nenalezeno" line.
- i18n cs + en in BOTH locale files (Czech default). No hardcoded UI text.
- Vitest tests: typing shows grouped results; clicking an album result navigates to its page; the search page renders entity sections when present.
