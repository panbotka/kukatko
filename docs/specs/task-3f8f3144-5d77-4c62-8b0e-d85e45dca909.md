# Import completeness verification tool + persist import failures

Today you cannot prove an import is complete: per-photo failures go only to
`slog.Warn`, and a run reports `status=done` even with hundreds of failed photos
(`internal/importer` `Counts` is four aggregate integers). The user explicitly wants a
tool that says "the import is complete and nothing is missing."

## Sources to reconcile against (verified production topology)
- **PhotoPrism** (`fotky.kotrzina.cz`) is the source of PHOTOS + files. Reconcile its
  photo count (and per-type counts, e.g. `type:raw` = 12, `type:video` = 6) and each
  photo's `Files[]` against what Kukátko imported — so a dropped RAW sibling is
  detected, not just a missing photo.
- **photo-sorter** (`sorter.kotrzina.cz`) is the source of VECTORS only (embeddings +
  faces, keyed by PhotoPrism UID). Its `GET /api/v1/stats` gives
  `total_photos` / `photos_with_embeddings` (~20,092) / `total_faces` (~112,806).
  Reconcile: does every Kukátko photo that has a photo-sorter embedding/faces actually
  carry them 1:1? List photos missing their vectors.

## Requirements

### 1. A completeness / reconciliation tool
- `kukatko import verify` (CLI) + a maintainer API endpoint + a surface in the import
  UI. It pulls the source totals above and reconciles them against Kukátko, reporting
  "source has N, Kukátko has M" AND a concrete **list of what is missing**: photo UIDs
  not imported, photos missing a file (e.g. a RAW sibling), photos missing their
  photo-sorter embedding/faces, albums/people/labels not transferred. Account for
  legitimate deltas (SHA256 dedup, skips) so the delta is explainable.

### 2. Persist import failures (make run status truthful)
- Record per-photo AND per-file import failures to a table (photo/file id, source,
  stage, error) instead of only `slog.Warn`; an import run with unresolved failures
  must NOT report `status=done`; surface the failures so they can be listed and
  retried. This closes the "satellites (markers/album membership/edits/pHash/files)
  are best-effort and silently lost" gap.

## Testing
- Integration tests: a verify run against a source with a known missing photo / missing
  RAW / missing embedding reports it; a run with induced per-photo failures is NOT
  `done` and lists them. `make check`. Update `docs/OPERATIONS.md` + `docs/API.md`.
