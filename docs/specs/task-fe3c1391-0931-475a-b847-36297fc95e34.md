# M4 — Bulk metadata editing API

Add endpoints to apply metadata changes to many photos at once.

## Context
Read `docs/ARCHITECTURE.md` §1 (hromadná úprava), §13. Depends on photos, albums, labels
repositories and the audit log. Operations must be transactional per batch and auditable.

## Requirements
- `POST /api/v1/photos/bulk` (editor/admin) accepting a list of photo UIDs + an operation set:
  - add/remove to/from album(s)
  - add/remove label(s)
  - set/clear description / caption
  - set/clear location (lat/lng)
  - set/clear private
  - archive/unarchive
  (Favorites are per-user — handle via the favorites API, but a bulk favorite/unfavorite for the
  current user is acceptable here too.)
- Apply within a transaction; return a per-photo result summary (updated/skipped/error) and
  overall counts. Validate the operation payload; reject unknown ops with 400.
- Write **audit log** entries (in the same transaction) describing the bulk change.
- Reasonable batch-size limit with a clear error when exceeded.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): bulk add-to-album + add-label + set-location across multiple
  photos; partial failures reported per-photo without aborting valid ones (or documented
  all-or-nothing semantics — pick one and test it); audit entries written; role enforcement;
  batch-size limit.
