# Fix "Back"/Esc from photo detail to return to the originating list

## Problem
When a user opens a photo's detail view from inside an album (Albums → an album → click a photo) and presses **Esc** (or clicks the header Back link, or uses browser back), they are returned to the **library** instead of back to that album. The same happens for labels/tags (štítky). The originating scope is lost.

## Root cause
The photo detail page reconstructs its "back" target from the `album`/`label`/`favorite` URL query params on `/photos/{uid}?…`. The library grid correctly injects those params into each tile's link, but the **album, label, favorites, and search** grids render `<PhotoGrid>` **without** passing the `detailQuery` prop, so their tiles link to a bare `/photos/{uid}` with no scope. `backHref` then falls through to the library. The prev/next neighbor paging on the detail page is unscoped for the same reason.

## Requirements
- Opening a photo from an **album** detail view and pressing Esc (or Back link, or browser back) returns to that **same album**, preserving that view's active state (filters/sort/page) where applicable.
- Same for a **label/tag** detail view: returns to that same label.
- Same for the **Favorites** view: returns to Favorites.
- Same for the **Search** results view: returns to the search (query + filters preserved).
- Prev/next navigation within the detail view, when the photo was opened from an album/label/favorites/search, pages **within that scope**, not the whole library.
- The Library view's existing correct behavior must not regress.

## Implementation notes
- The machinery already exists and works for the library: `detailQueryString(view)` builds the query, `PhotoGrid`/`PhotoTile` accept a `detailQuery` prop that becomes the tile link's query string, `readUrlState` reads it back on the detail page, and `backHref(view)` in `web/src/lib/detailView.ts` maps `view.album`/`view.label`/`view.favorite` to the right path.
- Reference implementation to copy: `web/src/pages/LibraryPage.tsx` (builds `detailQuery` and passes `<PhotoGrid detailQuery={…} />`).
- Fix by passing `detailQuery` to the grids in `web/src/pages/AlbumDetailPage.tsx`, `web/src/pages/LabelDetailPage.tsx`, `web/src/pages/FavoritesPage.tsx`, and `web/src/pages/SearchPage.tsx`, with the originating scope injected — e.g. album page `{ ...view, album: <albumUid>, label: '', favorite: '' }`, label page `{ ...view, label: <labelUid>, album: '', favorite: '' }`, favorites `{ ...view, favorite: 'true' }`, search its search view. Verify each against how `backHref`/`readUrlState` expect scope encoded.
- Esc handling in `web/src/pages/PhotoDetailPage.tsx` (Escape → `navigate(backHref(view))`) should NOT change — the fix is upstream in what the tile link carries.
