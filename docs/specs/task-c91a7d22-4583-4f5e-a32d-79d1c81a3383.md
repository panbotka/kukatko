# Main Navigation

The navbar (`web/src/components/Layout.tsx`) currently hides Albums and Labels inside a "Procházet" dropdown, while a Search link, a live search box, and a Saved-searches dropdown all compete for space. Rebalance it around how the library is actually browsed: by album, by label, by year.

## Requirements

- Albums (`/albums`) and Labels (`/labels`) become top-level, always-visible nav items.
- Remove the top-level Search link and the live search box from the navbar. Searching is reached from the library page and from `/search`.
- Remove the saved-searches dropdown from the navbar. Saved searches must remain fully usable, but only from the `/search` page: it gets an entry point to list, apply, and manage them, and `/saved` stays reachable from there.
- Keep a dropdown for the remaining browse destinations (Favorites, People, Places, Map). Keep the Tools and Admin dropdowns and their existing role gating.
- Every nav item and every clickable card link gets an icon plus a `title` attribute that describes the action, not the noun — for example "Zobrazit alba" rather than "Alba". The same users will use this daily, so the icons are there to speed up recognition.
- Use a single consistent icon set (bootstrap-icons fits the stack; add it as a dependency if it is not already present). Icons are decorative: mark them `aria-hidden` and keep the visible text label.
- Do not touch the language switcher in this task; a separate task moves it.
