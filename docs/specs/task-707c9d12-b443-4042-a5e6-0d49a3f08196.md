# M0 — Project scaffolding & quality gate

Set up the Go backend skeleton, build tooling, and the strict lint+test gate so that every
later task builds on a green `make check`.

## Context
Greenfield project. Read `docs/ARCHITECTURE.md` for the full system design (esp. §4 Tech stack,
§14 Build, §19 Quality/testing/linting). Backend is **Go, single static binary
(`CGO_ENABLED=0`)** that will serve an embedded React/Vite frontend via `//go:embed`. Module
path: `github.com/panbotka/kukatko`. Go 1.26.

## Requirements
- `go.mod` with module `github.com/panbotka/kukatko`, Go 1.26. HTTP router `go-chi/chi/v5`.
  CLI via `spf13/cobra`.
- Project layout: `cmd/kukatko/main.go` (Cobra root + `serve` subcommand), `internal/` for
  packages (e.g. `internal/server`, `internal/version`). Keep `main` thin.
- `kukatko serve` starts a chi HTTP server (port from a hardcoded default for now, e.g. 8080;
  config comes in a later task) exposing `GET /healthz` → 200 JSON `{"status":"ok","version":...}`.
  Graceful shutdown on SIGINT/SIGTERM.
- `kukatko version` prints version/commit (ldflags-injectable vars in `internal/version`).
- **`.golangci.yml`**: copy and adapt the strict golangci-lint **v2** config from the
  photo-sorter project at `/home/pi/projects/photo-sorter/.golangci.yml` (same linter set and
  settings: revive/gosec/errcheck/errorlint/wrapcheck/cyclop/gocognit/funlen/dupl/goconst/
  gocritic/prealloc/sqlclosecheck/bodyclose/noctx/testifylint/thelper/usetesting/nilerr/lll(120)/
  misspell/godot/nakedret/unparam/wastedassign/…). Remove photo-sorter-specific errcheck
  exclude-functions (they reference photo-sorter packages); keep the generic exclusions.
- **`Makefile`** with targets: `fmt` (gofmt/gofumpt), `vet` (go vet), `lint` (golangci-lint run),
  `lint-fix`, `test` (unit tests, `go test ./...` excluding integration), `test-integration`
  (runs integration tests requiring `KUKATKO_TEST_DATABASE_URL`), `check` (= fmt + vet + lint +
  test), `build` (will later build frontend then `go build`; for now just `go build ./...`).
- At least one real unit test (e.g. version/healthz handler test using `httptest`) so `make test`
  exercises the suite.
- `README` note or `docs/DEVELOPMENT.md` with how to run `make check` and `kukatko serve`.

## Quality gate (mandatory)
- Use the **golang-developer** skill for all Go code.
- `make check` MUST pass with the strict `.golangci.yml` (do not weaken it). Exported symbols
  documented, errors wrapped, functions small.
- Do not commit any secrets. Leave `.secrets/` untouched (already gitignored).

## Out of scope
Config loading, DB, frontend, auth — separate tasks. Keep this minimal but lint/test-clean.
