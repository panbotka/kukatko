# M1 — Video support: storage, ingest, poster/thumbnails, metadata

Extend the media pipeline to accept and process video files (PhotoPrism stores many, incl. live
photos / mp4), so videos live alongside photos in the library.

## Context
Read `docs/ARCHITECTURE.md` §3.2 (storage/ingest), §4 (image processing — extend to video), §5.
Keep CGO disabled — **shell out to `ffmpeg`/`ffprobe`** for video. Depends on storage, upload
pipeline, thumbnailer, EXIF, photos schema. Add `ffmpeg` to runtime/apt deps and CI.

## Requirements
- Migration extending `photos`: `media_type` TEXT CHECK(image|video|live) default 'image',
  plus video fields: `duration_ms` INT, `video_codec` TEXT, `audio_codec` TEXT, `has_audio` BOOL,
  `fps` DOUBLE PRECISION. (Live photos: store the still as the primary image and link the motion
  video file.)
- Ingest: accept common video types (mp4, mov, m4v, avi, mkv, webm, hevc/h264). Probe metadata via
  `ffprobe` (duration, codecs, dimensions, fps, creation time, GPS if present); fall back to
  exiftool. Store original via the storage layer (SHA256 dedup, `YYYY/MM`).
- **Poster + thumbnails**: extract a representative frame via `ffmpeg` (e.g. ~1s in or scene
  detection) and feed it through the existing thumbnailer to produce the same tile/fit sizes, so
  the grid shows a video poster. Mark tiles as video (duration overlay handled in UI tasks).
- **Searchability**: enqueue `image_embed` (and optionally `face_detect`) on the **poster frame**
  so videos participate in semantic/similar search and people.
- Detect missing `ffmpeg`/`ffprobe` and return a clear error.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass. Add `ffmpeg` to CI for integration.
- Integration tests (test DB) with a tiny embedded sample video: ingest creates a video photo row
  with media_type=video + probed metadata; poster frame extracted and thumbnails generated;
  embedding enqueued on the poster; SHA256 dedup works for videos. Unit-test the ffprobe/ffmpeg
  command construction + metadata parsing (skip actual ffmpeg if absent in unit suite).
