# M4 — Albums & labels UI

Build the UI to browse and manage albums and labels and to view their photos with filters/sort.

## Context
Read `docs/ARCHITECTURE.md` §13. Backend: albums/labels APIs + scoped photo listing. Reuse the
library grid + urlState convention. react-bootstrap (Superhero), i18n, responsive.

## Requirements
- **Albums page**: grid of albums (cover, title, count). Create/rename/delete (editor/admin),
  set cover, toggle private. Album detail = the photo grid scoped to the album, with
  filters/sort/pagination in the URL, plus reorder (drag) and add/remove photos.
- **Labels page**: list of labels with counts; create/rename/delete; clicking a label shows its
  photos in the grid (filters/sort in URL).
- Add-to-album / add-label affordances from the grid selection (wire to bulk API where natural).
- Empty/loading/error states; role-gated mutations (viewer read-only); i18n (cs/en).

## Quality gate (mandatory)
- `make check` MUST pass (frontend ESLint + Vitest).
- Vitest tests: album create/rename/delete flows call API and update UI; scoped grid reflects
  album/label in URL; reorder persists; viewer cannot see mutation controls. Mock API.
- Typed components; all text via i18n.
