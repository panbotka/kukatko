# Collapse the main navbar into grouped dropdown menus

The top navbar (`web/src/components/Layout.tsx`) currently renders ~13 flat links (Domů, Knihovna, Oblíbené, Alba, Štítky, Hledat, Lidé, Mapa, Místa, Nahrát, Duplikáty, Koš, Import, Údržba, Systém). It is overcrowded and hard to scan. Group related destinations into a few dropdown menus so the bar is clean and the information architecture is obvious.

## Requirements

- Reduce the number of top-level navbar entries by grouping related routes into `react-bootstrap` `NavDropdown` menus.
- Suggested grouping (refine wording, but keep the intent):
  - A **"Procházet" / Browse** dropdown: Knihovna, Oblíbené, Alba, Štítky, Lidé, Místa, Mapa.
  - Keep **Hledat** and **Nahrát** prominent (top-level or clearly reachable).
  - An **editor-only "Nástroje" / Tools** dropdown: Duplikáty, Koš.
  - An **admin-only "Správa" / Admin** dropdown: Import, Údržba, Systém.
  - Keep **Domů** reachable (brand/logo link is fine).
- Preserve ALL existing role gating (viewer/editor/admin). Items appear only for allowed roles; a dropdown whose children are all hidden must itself be hidden. Editor gate uses `canWrite`, admin gate uses `isAdmin` (as today).
- Parent menu shows an **active state** when any of its child routes is the current route.
- Add i18n keys for the new group labels in BOTH `web/src/i18n/locales/cs/common.json` and `.../en/common.json` (cs default). No hard-coded UI strings.
- Must remain fully usable on mobile: in the collapsed burger menu the groups expand inline, with large tap targets (reuse the `kukatko-tap-target` convention where relevant).
- Keep the existing navbar extras working unchanged: `NavbarSearch` (quick results), `SavedSearchesMenu`, `LanguageSwitcher`, `KeyboardShortcutsHelp`, and the user/logout menu.

## Edge Cases

- A viewer (read-only) sees no Tools/Admin dropdowns at all.
- Keyboard accessibility: dropdowns open/close with keyboard, focus states visible.
