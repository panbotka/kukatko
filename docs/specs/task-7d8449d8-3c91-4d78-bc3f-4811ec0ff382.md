# Maps: surface upstream tile failures instead of silent grey tiles

The map shows blank grey tiles with no explanation. The cause is **outside our code**: mapy.com currently answers our API key with `403 Forbidden` on both `/v1/maptiles` and `/v1/geocode` (verified with curl straight against the upstream — the response comes from `mapy-api-gateway`, our proxy is never at fault). A dead or over-quota key must not look like a broken app.

**This task does not fix the key** (that is a manual action in the mapy.com console). It makes the failure legible and cheaper.

## Requirements

- **Log it.** When mapy.com answers a tile or geocode request with a non-200, log a WARN carrying the upstream status and the mapset (`internal/mapy/mapy.go` already maps non-200 to `ErrUpstream` / `ErrUnavailable` at lines ~229 and ~280 — the status is currently swallowed).
- **Distinguish auth failures.** `401`/`403` from mapy.com (rejected/expired/over-quota key) must map to a distinct sentinel error, not the generic `ErrUpstream`, and the tile proxy must translate it to a distinct HTTP status the frontend can recognise (do not pass the raw upstream 403 through unchanged — it is our key that is bad, not the caller's request).
- **Tell the user.** The map view shows a dismissible warning ("Mapové podklady se nepodařilo načíst — mapový klíč byl odmítnut", i18n cs+en) instead of an unexplained grey grid. Photo markers/clusters still render on the empty background — the map page must stay usable.
- **The admin system status** (`internal/system` / `GET /system/status`) reports the maps backend as degraded when the key is being rejected, so this is visible without opening the map.
- **Server-side tile cache.** Tiles are only browser-cached (`defaultTileCacheMaxAge`, 24 h); every pan burns mapy.com credits (free tier = 250k/month, 1 tile = 1 credit) and this is the second time maps have died. Add a bounded server-side cache of proxied tiles so repeated views cost nothing. Cache successes only — never cache an error response.

## Implementation Notes

- `internal/mapy/mapy.go` (upstream client, `statusError`), `internal/mapsapi/mapsapi.go` (proxy handler, `handleTile`), `web/src/components/map/LeafletMap.tsx` + `web/src/services/map.ts` (tile layer, error surfacing).
- Keep the key server-side — it must never reach the browser, including in any error payload.
- Tests: fake the upstream (the mapy client is already behind an interface) and cover 403 → degraded status → UI warning, plus a cache hit not re-calling upstream.
