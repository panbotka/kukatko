# App-wide UI/UX audit + high-impact usability improvements

Do a whole-app UI/UX pass focused on **ordinary, non-technical users**: it must be obvious how to do everything, every interactive element must be big enough, and the app must feel simple and calm rather than dense. Produce an audit document AND apply a bounded set of safe, global, high-impact improvements in this same session. Do NOT attempt to rebuild every screen.

## Part 1 — Audit document

- Create `docs/UX_AUDIT.md`: a **screen-by-screen** review of every page in `web/src/App.tsx` routing (Login, Home, Library, Favorites, Albums + detail, Labels + detail, Search, Saved searches, People + subject + clusters, Map, Places, Photo detail, Upload, Trash, Duplicates, Import, Maintenance, System, Account).
- For each screen list concrete issues and **prioritized recommendations** (impact vs effort), organized around these lenses:
  - **Clarity** — is it obvious what each screen is for and how to act? Are primary actions prominent?
  - **Touch-friendliness** — every button/icon/link ≥ ~44px; adequate spacing.
  - **Consistency** — buttons, headers, cards, spacing, terminology consistent across screens.
  - **Readability** — font sizes, contrast, hierarchy.
  - **States** — empty / loading / error states present and friendly.
  - **Intimidation** — anything that would make a non-technical user hesitate.
- End the document with a **prioritized backlog** the team can turn into follow-up tickets.

## Part 2 — Bounded global quick wins (implement now)

Apply only safe, non-breaking, app-wide improvements that fit in one session, for example:

- Enforce consistent, large tap targets and spacing via shared styles (`web/src/styles/app.css`, the `kukatko-tap-target` convention).
- Consistent button sizing and clear primary-action prominence across pages.
- Consistent section/card headers and page titles.
- Clearer, friendlier empty states where missing.
- Fix obviously confusing labels/wording (update both cs and en i18n files).

Keep code changes conservative. If a recommendation is large or risky, **document it in the audit instead of implementing it**.

## Out of scope (tracked separately — reference in the audit, do NOT implement here)

- Navbar dropdown grouping.
- Library FilterBar redesign.
- Fullscreen photo viewer.
- Album/label add autocomplete.
- Map-based location picker.
