# M4 — Per-user favorites API

Expose endpoints for users to favorite/unfavorite photos and browse their favorites.

## Context
Read `docs/ARCHITECTURE.md` §5 (`user_favorites`), §1. Favorites are **per-user**. Depends on the
favorites repository and the photo-list machinery.

## Requirements
- `PUT /api/v1/photos/{uid}/favorite` and `DELETE /api/v1/photos/{uid}/favorite` for the current
  authenticated user (idempotent). Any authenticated role may favorite (it's personal).
- Include an `is_favorite` flag (for the current user) in photo detail and list responses, and
  support a `favorite=true` filter in the photo-list endpoint scoped to the current user.
- `GET /api/v1/favorites` convenience listing (current user's favorites) reusing the photo-list
  shape with filters/sort/pagination.

## Quality gate (mandatory)
- Use the **golang-developer** skill. `make check` MUST pass.
- Integration tests (test DB): favorite/unfavorite idempotent; per-user isolation (user A's
  favorites invisible to user B); `is_favorite` reflected in list/detail; favorite filter and
  `/favorites` listing correct.
