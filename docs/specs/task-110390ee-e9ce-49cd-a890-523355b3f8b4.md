# Saved Searches (Smart Albums) — Backend

Per-user saved searches: a named, owner-private filter/search definition the user can re-open later. Only the owner can see or modify their saved searches (mirror the per-user ownership model of `user_favorites`). This task is backend only.

## Requirements
- New migration under `internal/database/migrations/` using the next free number: table `saved_searches` — `uid` PRIMARY KEY (generate with a new short prefix, e.g. `ss`), `owner_uid` (FK users `ON DELETE CASCADE`), `name TEXT NOT NULL`, `params JSONB NOT NULL` (opaque saved view/search state: filters, sort, search query, mode), `created_at`/`updated_at TIMESTAMPTZ`. Index on `owner_uid`.
- New package `internal/savedsearch` with a `Store` over the shared pgx pool: `Create(ctx, ownerUID, name, params)`, `List(ctx, ownerUID)` (owner's searches, newest-first), `Get(ctx, uid)`, `Update(ctx, uid, name, params)`, `Delete(ctx, uid)`. Store `params` as raw JSON (e.g. `json.RawMessage`). Sentinel `ErrNotFound`.
- New package `internal/savedsearchapi`: `NewAPI(Config{Store, RequireAuth})` + `RegisterRoutes` mounting under `/api/v1`:
  - `GET /saved-searches` → the current user's saved searches `{saved_searches:[{uid,name,params,created_at,updated_at}]}`.
  - `POST /saved-searches` body `{name, params}` → 201 (empty name → 400).
  - `GET /saved-searches/{uid}` → 200; if the row belongs to another user, respond 404 (never reveal others' searches).
  - `PATCH /saved-searches/{uid}` body `{name?, params?}` → 200 (404 if missing or not owner).
  - `DELETE /saved-searches/{uid}` → 204 (404 if missing or not owner).
  - All endpoints require auth; every operation is scoped to the acting user (taken from the auth context). `DisallowUnknownFields` + a body size limit, consistent with the other APIs.
- Mount via a new `server.WithAPI` builder (e.g. `buildSavedSearchAPI`) in `cmd/kukatko/`.
- Integration tests: full CRUD plus owner isolation — user B must not be able to read, patch, or delete user A's saved search (expect 404).
