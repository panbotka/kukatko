# People detail gallery: images-per-row control + quieter cover button

Two small, related UX fixes to the People/Subject detail page (`/people/{uid}`, component `web/src/pages/SubjectPage.tsx`, gallery tile `web/src/components/people/SubjectPhotoTile.tsx`). Read `docs/FRONTEND.md` (grid density + subject page) first.

## Context

- The subject gallery already CONSUMES the shared grid density (`SubjectPage.tsx` uses `useGridDensity()` + `gridTemplateColumns(density, GALLERY_GAP_PX)`), but it never renders the control, so there is no in-page way to change how many photos appear per row. Every other gallery (Library, albums, labels, search, favorites) exposes the control via `FilterBar` → `web/src/components/library/GridDensityControl.tsx` (a compact −/+/Auto stepper, takes no props, reads/writes the same global `useGridDensity` localStorage state).
- Each subject tile shows an always-visible dark button overlaid at the bottom-left labelled **"Nastavit jako náhled"** (i18n key `subject.cover.set`; current-cover state key `subject.cover.current`) that sets the subject's cover photo (`PATCH /subjects/{uid}` with `cover_photo_uid`). It is repeated on every tile and is visually too loud. (This is the "needlessly big" button the change targets.)

## Requirements

1. **Add the images-per-row control** to the subject detail page: render `<GridDensityControl />` in the page header (near the Edit / Select buttons in the `SubjectPage.tsx` header row). It needs no props and immediately re-columns the existing subject grid via the shared state. It must be visible to all users who can see the page (not gated by write permission — it's a view preference). Optionally align the gallery gap to the shared convention so the people gallery matches other galleries visually.
2. **De-emphasize the "Set as cover" button** in `SubjectPhotoTile.tsx`: make it far less prominent instead of an always-on loud button on every tile. Reveal it on hover/focus of the tile (keep it keyboard-accessible), and/or reduce it to a small icon-only button (bootstrap-icons via the `Icon` component, e.g. a star/image icon) with an accessible label/tooltip. The current-cover state must still be indicated (e.g. a subtle filled icon or small badge). Behaviour is unchanged — same `onSetCover` handler and `PATCH /subjects/{uid}` — only visual prominence and reveal change.
   - Keep it working under the existing conditions (`canSetCover && !selectable`), keep it clear of the selection checkbox, and ensure it stays reachable on touch devices (no hover) — e.g. still show a small icon on touch, or surface it via the tile's existing controls.

## Tests

- Vitest: the subject page renders the density control and changing it re-columns the grid (assert `gridTemplateColumns`/`data-density` responds); the cover affordance is not a large always-visible button (assert it's hover/focus-revealed or icon-only) yet remains operable and calls the set-cover handler; the current-cover indicator shows for the cover photo.
