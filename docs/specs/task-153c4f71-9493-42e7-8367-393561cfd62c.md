# Multi-select & batch actions

Bring selection and bulk editing up to the standard of modern photo apps: a light hover
affordance to select, a clear selected state, and a floating action bar for batch operations.

## Requirements

- **Selection affordance:** each grid tile shows a circular checkmark control in a corner on
  hover (and always, once any item is selected). Clicking it toggles selection without opening
  the photo.
- **Selected state:** a selected tile gets an accent ring and a filled check; the image dims
  slightly so selection is unmistakable across a dense grid.
- **Range select:** Shift-click selects the contiguous range between the last selected tile and
  the clicked one. Provide a "select all / clear" affordance and a live selection count.
- **Floating batch action bar:** when ≥1 item is selected, a bar slides up from the bottom
  (over the content, using a token surface + blur) showing the count and batch actions — add to
  album, add/remove label, favorite, archive, download — wired to the existing bulk endpoint
  (`POST /photos/bulk`). The whole batch is applied in one request.
- **Esc clears the selection** and hides the bar. Selection is transient (not persisted across
  navigation).
- Success/failure surface as a toast; a failed batch leaves selection intact so it can be
  retried.
- Icons via the `Icon` component; design tokens for the bar surface, accent ring and focus
  ring. i18n (cs/en) for all new strings.

## Verification

- Drive it in a browser against a throwaway instance: hover-select, shift-range-select, run a
  batch action, confirm the toast and the resulting change.
- `make check` must pass.
