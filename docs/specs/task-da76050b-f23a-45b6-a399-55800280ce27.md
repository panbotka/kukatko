# M6 — Library maintenance: integrity scan & repair

Add maintenance tooling to keep a large, long-lived library consistent: detect disk↔DB drift and
backfill/regenerate derived data.

## Context
Read `docs/ARCHITECTURE.md` §3.2 (storage), §15. Depends on storage, photos repository,
thumbnailer, job queue, embeddings/faces backfill endpoints. Mirrors photo-sorter's
`cache build-thumbs` but broader.

## Requirements
- `kukatko maintenance` command + admin endpoints, covering:
  - **Integrity scan**: report photos whose original file is missing on disk; originals on disk not
    referenced in DB (orphans); photos missing thumbnails; photos missing embeddings/faces.
    Output a structured report (counts + samples).
  - **Repair actions** (each opt-in): regenerate missing thumbnails; enqueue `image_embed`/
    `face_detect` for photos missing them; optionally import orphan originals on disk into the DB;
    recompute pHash/dHash where missing.
  - Run repairs through the job queue with bounded concurrency; resumable; progress reporting.
- An admin "Maintenance" UI section to run a scan and trigger repairs with progress (can reuse the
  jobs stats polling). i18n, admin-only.
- Safe and idempotent; never deletes originals (that's the trash/purge task's job).

## Quality gate (mandatory)
- Use the **golang-developer** skill; `make check` MUST pass (incl. frontend lint/test).
- Integration tests (test DB + temp storage): scan detects missing-original, orphan-file,
  missing-thumb, missing-embedding cases; regenerate-thumbnails repair works; backfill enqueues
  only missing ones; idempotent re-run. Vitest for the maintenance UI trigger/progress (mock API).
