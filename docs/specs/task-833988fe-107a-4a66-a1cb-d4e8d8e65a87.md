# Keyboard Shortcuts — UI

Add keyboard-driven navigation and actions to the photo grid and detail view, with a discoverable help overlay. Frontend only. (Numeric rating hotkeys are handled by the ratings UI task — do NOT add rating keys here.)

## Requirements
- A `useKeyboardShortcuts` hook plus a small shortcut registry, used by `LibraryPage`/`PhotoGrid` and `PhotoDetailPage`.
- Grid (`PhotoGrid` on `LibraryPage`): arrow keys and `j`/`k`/`h`/`l` move a visible focus highlight between tiles (focus must follow virtualization and scroll the focused tile into view). `Enter` opens the focused photo's detail. `x` toggles selection of the focused tile (enabling selection mode if it is off, integrating with the existing `useSelection`). `f` toggles favorite on the focused tile (reuse `useFavorite` / the favorite service). `Escape` clears selection, then focus.
- Detail (`PhotoDetailPage`): keep existing `←`/`→` prev/next. Add `f` to toggle favorite and `Escape` to go back to the source view.
- A help overlay opened with `?` (Shift+/) listing all shortcuts grouped by context (Grid / Detail), dismissed with `Escape` or a close button. Provide a small keyboard-icon button that also opens it.
- Never fire shortcuts while the user is typing in an input/textarea/contenteditable, or while a modal containing a form is open.
- Preserve all existing click/touch behavior and accessibility — focus states must be clearly visible.
- i18n cs + en in BOTH locale files for the help overlay text (Czech default).
- Vitest tests: arrow keys move focus, `Enter` opens detail, `x` selects the focused tile, `?` opens the help overlay, and shortcuts are suppressed while typing in an input.
