# Remote CLI: The Rest Of The Surface

`kukatko ctl` can already talk to a remote instance, resolve a context, authenticate with a bearer token, and drive the `photos` commands. Extend it to the rest of the surface worth operating from a terminal or by an AI agent.

## Commands

- `ctl albums list | get <uid> | create | add-photos | remove-photos`
- `ctl labels list | get <uid> | create | attach | detach`
- `ctl subjects list | get <uid>` plus the subject's photo gallery
- `ctl favorites list | add <uid> | remove <uid>`
- `ctl rating set <uid> <0-5>`
- `ctl bulk` — apply a bulk metadata edit to a set of photo UIDs read from arguments or from stdin, so it composes with `ctl photos list -o json`

## Deliberately excluded

Backup, restore, migration, maintenance, import, and the job queue stay local-only commands. They are destructive or long-running and are not worth exposing over the network. Do not add them.

## Requirements

- Reuse the client, context resolution, authentication, and output formatting already built for `ctl photos`. Do not fork them.
- The API has **no uniform list envelope**. `photos` returns `{photos, total, limit, offset, next_offset}`, but `albums` returns a bare `{albums: [...]}` and `labels` a bare `{labels: [...]}`, neither paginated. Write a decoder per resource. Do not normalise the API to make a generic decoder possible — the frontend depends on the current shapes.
- Mutating commands respect the server's role checks: a viewer's token gets a `403`, and the message says so plainly rather than dumping the response body.
- `ctl bulk` must send one request for the whole batch, matching the server's single-transaction contract on `POST /photos/bulk`. Do not loop per photo.
- A command that would affect more than 50 photos asks for confirmation unless `--yes` is passed.
- Keep the default output a compact table; `-o json` passes the API's JSON through unchanged.

## Verification

`make check` must pass. Add tests against an `httptest` server for each resource's list and get, for the bulk command's single-request behaviour and its confirmation gate, and for the `403` message a viewer sees. Update `docs/OPERATIONS.md` with the full command tree.
